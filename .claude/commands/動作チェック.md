# アプリ動作チェック（シナリオ QA）

台本と実アプリの数値整合性、アプリ内の整合、デモデータの現実性・教育性を自動チェックします。

## 実行内容

以下を順に実行（Makefile 経由）:

```bash
make qa
```

これは以下の 5 ステップを包含:
1. `qa:claims` — 4 台本 MD から数値主張 61 件を抽出 → `reports/scenario_claims.json`
2. `qa:playwright` — DOM から実値を取得し、台本と照合 → `reports/scenario_qa_playwright.json`
3. `qa:internal` — アプリ内整合性 pytest（サマリー ↔ カンファ Block A の一致等）
4. `qa:realism` — デモデータ現実性・教育性 pytest
5. `qa:report` — 統合レポート生成 → `reports/scenario_qa_report_YYYY-MM-DD.md`

## 実行手順

1. `make qa` を Bash で実行
2. 出力結果から以下を抽出して副院長に報告:
   - 🔴 重大不一致の件数と内容
   - 🟡 軽微ズレの件数
   - 🟢 デモデータ現実性（OK / NG）
   - ⚠️ 教育性（推奨調整）
   - 全体の pass/fail
3. 重大不一致があれば、修正案（台本側を直すか、アプリ側を直すか）を提示
