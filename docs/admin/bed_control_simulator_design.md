# ベッドコントロールシミュレーター 設計ドキュメント

> **対象施設**: おもろまちメディカルセンター 地域包括医療病棟（94床）
> **作成日**: 2026-03-25
> **ステータス**: 設計フェーズ

---

## 1. 設計の要約

### 目的

地域包括医療病棟（94床）において、以下の運営目標を同時に達成するための **病棟運営支援ツール** を構築する。

| 運営目標 | 数値ターゲット |
|:---|:---|
| 病床稼働率 | 90〜95% |
| 平均在院日数 | 17日前後（最適ゾーン） |
| 月間入院数 | 約150件/月 |
| 運営貢献額の最大化 | A/B/Cフェーズ構成比の最適制御 |

### スコープ

- **含む**: 日次ベースの患者フロー・稼働率・診療報酬収入シミュレーション、3戦略の比較、運営改善アラートの自動出力
- **含まない**: 診療報酬の厳密計算（DPC点数テーブル等）、個別患者の重症度評価、看護必要度の算定シミュレーション

> **注意**: 本シミュレーターは病棟運営判断用の簡易モデルであり、診療報酬の厳密計算を目的としない。

## 2. モデル化の前提と限界

### 2.1 患者フェーズモデル

患者を在院日数に基づき3フェーズに分類する。コストモデルは運営貢献額ベース（変動費のみ）。

```
入院 ──→ [A群: 1〜5日目] ──→ [B群: 6〜14日目] ──→ [C群: 15日目〜] ──→ 退院
           変動費高・運営貢献額小    変動費低・運営貢献額大    変動費最小・運営貢献額中
```

| フェーズ | 在院日数 | 日次診療報酬 | 日次変動費 | 日次運営貢献額 | 特徴 |
|:---:|:---:|---:|---:|---:|:---|
| **A群** | 1〜5日目 | 36,000円 | 12,000円 | **24,000円** | 検査・処置集中期。変動費高 |
| **B群** | 6〜14日目 | 36,000円 | 6,000円 | **30,000円** | 安定期。最も運営貢献額を生む |
| **C群** | 15日目〜 | 33,400円 | 4,500円 | **28,900円** | 回復・退院準備期。退院調整の柔軟性が高い層 |

### 2.2 前提条件

- 包括払い制度のため日次診療報酬は一定（30,000円/日）と仮定
- 入院はポアソン分布に近い日次変動あり
- 退院は在院日数に応じた確率的モデル（シグモイド関数ベース）
- C群は退院調整の柔軟性が高い層として機能（退院タイミングを戦略的に制御可能）
- 月間入院数は約150件/月を基準とする

### 2.3 モデルの限界

| 限界事項 | 影響 | 対応方針 |
|:---|:---|:---|
| 個別患者の重症度・疾患を考慮しない | フェーズ別コストの精度低下 | 平均値で代替 |
| 看護必要度の算定への影響が未反映 | 施設基準維持との連動不可 | 今後の改善で対応 |
| 季節変動は変動係数で簡易表現 | 繁忙期/閑散期の精度限界 | 実データ蓄積後に改善 |
| 緊急入院と予定入院の区別なし | 入院タイミング制御の精度低下 | 将来的に分離 |
| 実際の診療報酬点数とは異なる | 報酬予測の絶対値に誤差 | 相対比較に活用 |

## 4. 計算ロジック

### 4.1 日次更新ロジック（メインループ）

各シミュレーション日について、以下の4ステップを順次実行する。

```
┌─────────────────────────────────────────────────┐
│                 日次更新ループ                      │
│                                                   │
│  Step 1: フェーズ進行                               │
│  全在院患者の在院日数を +1                           │
│  フェーズ判定を更新（A→B→C）                        │
│         │                                         │
│         ▼                                         │
│  Step 2: 退院判定                                   │
│  戦略に応じた退院確率で退院処理                       │
│         │                                         │
│         ▼                                         │
│  Step 3: 新規入院                                   │
│  空床数と戦略に応じて新規入院を受け入れ               │
│         │                                         │
│         ▼                                         │
│  Step 4: 指標計算                                   │
│  稼働率・診療報酬収入・コスト・運営貢献額・構成比を算出             │
└─────────────────────────────────────────────────┘
```

### 4.2 退院モデル

#### 基本退院確率（シグモイド関数）

在院日数 `d` に対する基本退院確率 `P_discharge(d)` を以下で定義する。

```
P_discharge(d) = 1 / (1 + exp(-k × (d - m)))
```

- `k`: 急峻度パラメータ（`discharge_sigmoid_steepness` = 0.3）
- `m`: 中点パラメータ（`discharge_sigmoid_midpoint` = 17日）

#### 退院確率の在院日数別推移（初期値での例）

| 在院日数 | 基本退院確率 | 解釈 |
|:---:|:---:|:---|
| 5日目 | 2.7% | A群末期。ほぼ退院しない |
| 10日目 | 10.9% | B群前半。退院は少数 |
| 14日目 | 28.9% | B群末期。約3割が退院 |
| 17日目 | 50.0% | 中点。半数が退院判定 |
| 21日目 | 76.9% | C群。大半が退院方向 |
| 30日目 | 98.2% | ほぼ確実に退院 |

#### 退院調整ロジック

- **退院準備**: 退院予定日の `discharge_prep_days`（2日）前から退院確率を1.2倍に補正
- **強制退院**: `max_los`（60日）到達で退院確率を1.0に設定
- **戦略補正**: 各戦略の退院倍率を乗算（詳細は4.5節）

### 4.3 入院モデル

#### 日次入院数の生成

```
daily_admissions = Poisson(lambda)

lambda = (monthly_admissions / 30) × seasonal_factor × (1 + N(0, admission_variability))
```

- ポアソン分布で日次変動を表現
- `seasonal_factor` で季節変動を調整
- `admission_variability` で日内変動の幅を制御

#### 受入判定

```
if 在院患者数 >= total_beds:
    受入不可 → 未活用病床コストとしてカウント
elif 在院患者数 >= total_beds × 戦略別閾値:
    受入抑制（入院数を制限）
else:
    全数受入
```

### 4.4 診療報酬収入・コスト・運営貢献額の計算

#### 日次診療報酬

```
日次診療報酬 = Σ(各群患者数 × revenue_per_day)
         + 当日入院患者数 × admission_day_bonus
         + 在院14日以内患者数 × within_14days_bonus
         + 全患者数 × rehab_revenue
```

#### 日次コスト

```
日次コスト = A群患者数 × cost_phase_a
           + B群患者数 × cost_phase_b
           + C群患者数 × cost_phase_c
```

#### 日次運営貢献額

```
日次運営貢献額 = 日次診療報酬 - 日次コスト
```

#### 未活用病床コスト

```
未活用病床コスト = 満床超過需要数 × opportunity_cost
```

#### 稼働率

```
稼働率 = 在院患者数 / total_beds
```

### 4.5 3戦略の退院・受入判定ロジック

#### 戦略比較サマリ

| 項目 | 回転重視 | 安定維持 | バランス |
|:---|:---|:---|:---|
| **想定場面** | 繁忙期・入院待ち多数 | 閑散期・稼働率低下時 | 通常運用 |
| **C群退院倍率** | 1.5倍 | 0.7倍 | 1.0倍（条件付き変動） |
| **受入上限稼働率** | 97% | 93% | 95% |
| **重視指標** | 回転率・新規入院数 | 稼働率維持 | 構成比バランス |

---

#### 回転重視戦略

**目的**: 新規入院を最大化し、B群患者を効率的に確保する。

```
退院判定:
  C群退院確率 = 基本確率 × 1.5
  B群後半（12日目以降）退院確率 = 基本確率 × 1.2

受入判定:
  稼働率 <= 97% → 全数受入
  稼働率 > 97% → 受入停止
  空床があれば積極的に新規入院
```

**期待効果**: 平均在院日数の短縮、新規入院数の増加、A群比率の上昇
**リスク**: A群コスト負担増、退院調整の負荷増大

---

#### 安定維持戦略

**目的**: 稼働率を安定的に維持し、空床リスクを最小化する。

```
退院判定:
  C群退院確率 = 基本確率 × 0.7
  稼働率 < 85% → 全群退院確率をさらに × 0.8

受入判定:
  稼働率 <= 93% → 受入可
  稼働率 > 93% → 受入抑制開始
```

**期待効果**: 稼働率の安定、C群運営貢献額の確保
**リスク**: 新規入院の未活用病床コスト、在院日数の長期化

---

#### バランス戦略

**目的**: A/B/C群の構成比を最適ゾーンに維持し、総運営貢献額を最大化する。

```
退院判定:
  B群構成比 < 30% → 退院を抑制（全群 × 0.8）
  C群構成比 > 30% → C群退院促進（× 1.3）

受入判定:
  A群構成比 > 35% → 新規入院を抑制
  稼働率 90〜95% → 通常受入
  稼働率 < 90% → 積極受入
  稼働率 > 95% → 受入抑制
```

**期待効果**: 構成比の最適化による運営貢献額の最大化
**リスク**: 制御パラメータの調整が複雑

### 4.6 病棟運営支援フラグのロジック

日次で以下のフラグを自動判定し、アラートとして出力する。

| フラグ名 | 判定条件 | アクション示唆 |
|:---|:---|:---|
| **稼働率低下** | 稼働率 < 0.90 | 入院促進、退院抑制の検討 |
| **稼働率過多** | 稼働率 > 0.95 | 退院促進、新規入院抑制の検討 |
| **A群過多** | A群構成比 > 0.35 | 新規入院を一時抑制 |
| **B群不足** | B群構成比 < 0.25 | 退院抑制でB群滞留を確保 |
| **C群滞留** | C群構成比 > 0.35 かつ 空床 = 0 | C群の退院促進を優先 |

#### 推奨アクション数の算出

```
退院促進推奨人数:
  excess = 在院患者数 - (total_beds × target_occupancy_max)
  if excess > 0:
    C群から優先的に excess 人を退院候補としてリストアップ

保持許容人数:
  capacity = (total_beds × target_occupancy_max) - 在院患者数
  if capacity > 0:
    あと capacity 人まで受入可能
```
