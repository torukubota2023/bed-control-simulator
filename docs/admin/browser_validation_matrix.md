### 凡例
- ✅ : 動作確認済み（または公式サポート範囲内）
- ⚠️ : 一部機能に制限あり、要検証
- ❌ : 非対応（動作しない）
- 未検証 : テスト未実施

## 2. 検証手順

### 手順1: browser_probe.html によるブラウザ機能チェック

`tools/browser_probe.html` は、ブラウザの JavaScript API 対応状況を自動チェックするツールである。

**使い方:**

1. `tools/browser_probe.html` を検証対象の端末にコピーする（USBメモリ or 共有フォルダ経由）
2. 検証したいブラウザで `browser_probe.html` を開く（ファイルをダブルクリック）
3. 自動的に以下の項目がチェックされる:
   - JavaScript API の対応状況（`Array.at()`, `Object.hasOwn()` 等）
   - CSS 機能の対応状況
   - WebSocket 対応
4. 結果が画面に表示される（緑: 対応、赤: 非対応）
5. **結果のスクリーンショットを保存する**（SE確認用）

### 手順2: Streamlit アプリの動作確認

1. サーバーPCで Streamlit アプリを起動する
   ```
   streamlit run scripts/bed_control_simulator_app.py --server.address 0.0.0.0 --server.port 8501
   ```

2. 検証対象のブラウザで `http://[サーバーIP]:8501` を開く

3. 以下の項目を順番に確認する:

| # | 確認項目 | 確認方法 | 結果 |
|---|---------|---------|------|
| 1 | 初期表示 | ページが白画面にならず表示されるか | □ OK / □ NG |
| 2 | パスワード認証 | パスワード入力 → ログインできるか | □ OK / □ NG |
| 3 | サイドバー | セクション切替が動作するか | □ OK / □ NG |
| 4 | タブ切替 | 各タブをクリックして内容が切り替わるか | □ OK / □ NG |
| 5 | グラフ表示（pyplot） | matplotlib グラフが表示されるか | □ OK / □ NG |
| 6 | グラフ表示（plotly） | plotly インタラクティブグラフが動作するか | □ OK / □ NG |
| 7 | データ入力 | st.data_editor でデータ編集ができるか | □ OK / □ NG |
| 8 | シミュレーション実行 | ボタン押下 → 結果が表示されるか | □ OK / □ NG |
| 9 | CSV保存 | データの保存・読み込みが動作するか | □ OK / □ NG |
| 10 | HOPE連携 | メッセージ生成が動作するか | □ OK / □ NG |

### 手順3: 結果の記録

検証結果は以下のフォーマットで記録する:

```

browser_probe.html 結果: 全パス / ○項目失敗
アプリ動作確認: 全項目OK / 項目○でNG

備考:
```

## 3. 判定基準

| 判定 | 条件 | 対応 |
|------|------|------|
| 合格 | 全10項目 OK | そのまま展開可能 |
| 条件付き合格 | 項目7（data_editor）のみ NG、他 OK | フォールバック対応を検討 |
| 不合格 | 項目1〜6 のいずれかが NG | 別ブラウザでの再検証が必要 |
