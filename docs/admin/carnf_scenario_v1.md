# 多職種退院調整・連休対策カンファ 運用台本 v1

**作成日:** 2026-04-18
**対象アプリ:** ベッドコントロールシミュレーター v3.6（`scripts/bed_control_simulator_app.py`）
**画面:** サイドバー「🏥 退院調整」→「🏥 カンファ資料」タブ
**想定参加者:** 病棟師長・主任看護師・リハ PT/OT/ST・退院支援看護師（MSW）・担当医・副院長（任意）
**想定所要時間:** 20〜30 分／週（通常モード）、40〜60 分／大型連休前（連休対策モード ON 中）
**想定場面:** 病棟詰め所の据置 PC の前に 4〜6 名が立って／座って画面を囲む

---

## この台本の使い方

- 「何を画面で見て」「誰が発言して」「誰が操作して」「何が記録される」を時系列で書いています
- 参加者は台本を読まずに済むよう、司会（通常は師長）だけがこのファイルを開いておけば回せます
- 病棟ごとに運用が違う場合は、自院の呼び方・時間帯に合わせて章ごとの文言を調整してください
- 「操作：」の行はアプリの操作、「発言例：」の行は口頭でのやり取り、「画面上の変化：」の行は UI 側の反応を示します

---

## このカンファの 4 つの原則

1. **一画面で完結する** — 画面スクロールなしで 16:9 に収まるよう設計されている。スクリーンショット 1 枚で会議録代わりになる
2. **誰でも操作できる** — ステータスタグをクリックするだけでよく、PC 操作に慣れていないメンバーでも 30 秒で覚えられる
3. **記録は自動で残る** — ステータス変更は即座に JSON 保存され、次回カンファで前回の続きから開始できる
4. **個人情報は現場内で完結する** — 患者名・ID の編集フィールドは任意で、入力しても Git には上がらず院内 PC にのみ保存される

## 第0章：カンファ開始前の準備（5 分）

### 誰が：司会（通常は病棟師長）

### 操作の流れ

1. 詰め所の据置 PC を起動
2. ブラウザのブックマークから「ベッドコントロールシミュレーター」を開く

- 画面上部に「🗓 多職種退院調整・連休対策カンファ」というタイトルが表示される
- その右隣に「通常モード」または「🚨 連休対策モード」の色違いバッジ
- 右上に「2026-04-18 (金) | 5F 表示中 | （連休カウントダウン）」

### 病棟切替とモード切替

- ヘッダー直下の上部コントロール左側に「病棟」プルダウン（5F / 6F）
- その右に「連休対策モード」トグルスイッチ

**発言例（司会）：**

## 第1章：今月の目標達成進捗を全員で共有（2〜3 分）

### 見る場所：画面中央上部の KPI 横並び（ブロック A）

画面には 5 つの指標が並んで表示される：

| 位置 | 指標 | 何を意味するか |
|------|------|--------------|
| 1 | 今月の稼働率 | 今月の実績 vs 目標（全病棟 全月 一律 90%） |
| 2 | 平均在院日数（3ヶ月） | rolling 3 ヶ月の ALOS。制度上限 21 日が基準 |
| 3 | 救急搬送後 15% | 地域包括医療病棟の施設基準。未達だと減算 |
| 4 | 残り診療日 | 今月の残り日数 + 必要な床日 |
| 5 | 次の大型連休 | カウントダウン表示（近いほど強調色） |

### 発言例（司会）

> 「今月の稼働率は 85%、目標 90% に対してマイナス 5 ポイントです。残り 9 診療日で必要床日 28。ALOS は 17.5 日で制度上限 21 日までまだ余裕があります。救急搬送 17% で基準達成」
>
> 「次の大型連休は GW、あと○日です。21 日を切ると画面上部に推奨バナーが自動で出るので、それを合図に連休対策モードへ切り替えます」

### なぜこの共有が必要か

- 「稼働率が低い」だけでは対策が決まらない
- 「稼働率 85%・ALOS 余裕あり・救急達成」なら、**延ばすより新しい患者を受け入れる** 方針
- 「稼働率高い・ALOS 21 日近い・救急ぎりぎり」なら、**連休前退院を加速** する方針
- この 30 秒の状況確認で、今日のカンファのトーンが決まる

## 第2章：今週末・連休の見通しを確認（2〜3 分）

### 見る場所：画面中央左側（ブロック B）

ブロック B に表示されるもの：

- 通常モード：金・土・日の 3 日分の空床予測と救急余力
- 連休対策モード：連休 5 日分（例：4/29 水〜5/3 日）の空床予測と救急余力
- 下段に「週末受入余力 XX 床日（応需可能だった患者機会）」
- さらに下にステータス内訳（後述のブロック C で更新される値が自動集計される）

### 発言例（司会）

**通常モード時：**
> 「5F は今週末、金 4 床・土 8 床・日 7 床の空床見込み。土日の救急余力がマイナス 1、ほぼ埋まる前提です。応需可能だった患者機会は合計 19 床日」

**連休対策モード時：**
> 「GW 5 日間で 5F の空床見込み合計 66 床日。連休中は入院がほぼ止まるので、連休前にどれだけ退院できるかがこの 66 床日を何床日まで減らせるかに直結します」

### 用語の意図

- **受入余力**という言葉を使います。「空床コスト」「損失」という経営用語は避け、「受け入れられた可能性のある患者数」として可視化しています
- **応需可能だった患者機会**は、医療倫理上の観点から「空いていた床を金額で語らない」ための表現です。理事会向けでは金額換算を併記しますが、カンファ現場では床日ベースで話します

## 第3章：個別患者のカテゴリ分け（10〜15 分・本番）

### 見る場所：画面中央右側（ブロック C）・ここが議論の中心

画面に最大 10 名の患者行が表示される。在院日数の降順。各行の構成：

| 列 | 内容 | 備考 |
|---|------|------|
| 医師 / 患者 | 主治医の姓（サンプル） + 患者名・ID（任意編集） | 患者名・ID は空欄でも運用可 |
| 病棟 | 5F / 6F | 色分け表示 |
| 在院 | Day N | 在院日数 |
| 退院予定 | 日付または「未定」 | サンプル値。編集は別機能 |
| 確認事項 | 短文（リハ・MSW・主治医へのメモ） | 事前に入力済み |
| ステータス | クリッカブルタグ（現在の分類） | ★ ここがメイン操作 |
| 編集 | ✏️ ボタン | 主治医名・患者名・患者 ID 編集用 |

### 7 つのステータス（通常モード）

| タグ | 絵文字 | 意味 | 誰が発言のトリガーか |
|------|-------|------|-------------------|
| 方向性未決 | ⚫ | 退院先も退院時期も決まっていない | 医師または MSW |
| 医学的 OK 待ち | 🔵 | 主治医の退院判定が残っているだけ | 主治医 |
| 家族希望待ち | 🟢 | 医学的には退院可、家族の迎え日待ち | MSW・病棟看護師 |
| 施設待ち | 🟡 | 老健・特養・療養病床の受入待ち | MSW |
| 介護保険待ち | 🟣 | 認定調査・担当者会議・ケアマネ調整中 | MSW |
| リハ最適化中 | 🟠 | 機能目標に向けてリハ継続中 | リハ PT/OT/ST |
| 新規 | 🆕 | 初回カテゴライズ前 | 全員 |

### 4 つのステータス（連休対策モード）

| タグ | 絵文字 | 意味 |
|------|-------|------|
| 連休前退院 確定 | ✅ | 搬送・迎え・受入先すべて確定済 |
| 連休前退院 調整中 | 🛠 | 調整アクションが残っている |
| 連休中継続ケア | 🏥 | 連休中病院で過ごす方針確定 |
| 新規 | 🆕 | 未カテゴライズ |

### カテゴリ分けの判断基準（ここが議論の肝）

- **⚫ 方向性未決 vs 🔵 医学的 OK 待ち** の違い
  - 「主治医の OK が出れば退院先が見える」なら 🔵
  - 「退院先候補すら絞れていない」なら ⚫
  - この区別が曖昧だと次週のカンファで同じ議論を繰り返すので、30 秒迷ったら ⚫ に寄せる

- **🟢 家族希望待ち vs 🟡 施設待ち**
  - 自宅退院予定 → 🟢
  - 施設転院予定 → 🟡
  - 決まっていない → ⚫

- **🟠 リハ最適化中** は「退院可能性はあるが、もう少しリハで ADL を上げた方が長期予後に有利」と判断した場合のみ
  - 単純な延長希望ではない。「退院時機能を 1 点上げると 1 年死亡リスクが下がる」というエビデンスに基づく判断

### 操作の流れ（参加者全員で）

**発言例（師長）：**
> 「じゃあ上から順に見ていきます。伊藤医師の患者 Day 35、退院予定未定。4/24 までに田中医師が退院目処を再評価、要再カンファとなっています。今日のカテゴリどうしましょう」

**発言例（MSW）：**
> 「退院先、まだ候補出せていません。方向性未決ですね」

**操作（師長が PC で）：**
1. その患者行の「ステータス」列の 🆕 新規タグをクリック
2. ポップオーバーが開く
3. ラジオボタンで「⚫ 方向性未決」を選択
4. ポップオーバー外をクリックするか、そのまま次のステータスへ進む

**画面上の変化：**
- タグの色・絵文字が即座に ⚫ 方向性未決 の黒タグに変わる
- 画面左の「在院患者ステータス内訳」の「⚫ 方向性未決」カウンタが +1 される
- 裏側では `data/patient_status.json` に自動保存される（次回起動時も復元される）

### カテゴリ分けが進むにつれて起きること

- **ブロック B の内訳ブロックが自動更新される**
  - 「⚫ 方向性未決 3 名、🔵 医学的 OK 待ち 2 名、🟢 家族希望待ち 1 名…」のように刻々と変わる
  - カンファ終盤にはどのステータスに何名いるかが一目で分かる
- **ブロック D の職種別お願いが連動して変わる**
  - 後述

### 10 名全員のカテゴリ分けが終わったら

**発言例（師長）：**
> 「⚫ 未決 3 名、🟡 施設待ち 2 名、🟠 リハ最適化中 2 名、🔵 医学的 OK 待ち 1 名、🟢 家族希望待ち 1 名、🆕 新規 1 名。この未決 3 名が来週のキーですね」

## 第4章：職種別の今週のお願い（3〜5 分）

### 見る場所：画面下段（ブロック D、4 列レイアウト）

ブロック C でステータスが更新されるごとに、ブロック D の 4 役割（リハ・退院支援 NS・医師・看護）の今週の依頼事項が自動的に変わる。

例えば通常モードで 🟠 リハ最適化中が 2 名いれば：

- **🏃 リハ PT/OT/ST** 欄に「🟠 リハ最適化中 2 名：機能目標進捗を本日確認」と自動表示される
- **📋 退院支援 NS** 欄に「⚫ 未決 3 名：ADL 評価後に施設候補検討」と表示される
- **👔 医師** 欄に「🔵 OK 待ち 1 名：金朝カンファで判定」「⚫ 未決 3 名：退院目処の再評価」と表示される
- **🧑‍⚕️ 看護** 欄には週末処方確認・痛みコントロール再チェック等の定型項目

### 発言例（師長）

> 「職種別お願いを読み合わせます。リハは機能目標 2 名の進捗確認、退院支援 NS は未決 3 名の ADL 評価、先生方は OK 待ち 1 名の金朝判定と未決 3 名の再評価。看護は週末処方と引継ぎ。全員よろしいですか」

### この仕組みの狙い

- カンファで決まったことが、そのまま「誰が何をするか」に翻訳される
- 師長がカンファ後にタスクを洗い出す作業が不要になる
- 印刷すれば、その 1 枚がそのまま行動計画書になる

## 第5章：エビデンスの共有（1 分・毎回 or 隔週）

### 見る場所：画面最下部（紺色のファクトバー）

1 行 40〜80 文字のエビデンスが毎カンファで 1 件ローテーション表示される。

**表示例：**
> **エビデンス** 87 歳超でも多要素運動で Barthel が対照群 -5.0 → 介入群 +1.9 へ逆転 — 超高齢でも機能改善は可能 　Martínez-Velilla 2019 | JAMA Intern Med | PMID: 30419096

### 運用

- 病棟 / モードに応じた 20〜40 件のプールから、weight 付きでランダム抽選される
- 印刷時には自動的に非表示になる（`@media print { display:none }`）ので紙面を汚さない
- カンファ冒頭で司会が「今週の一言」として 15 秒読み上げる運用でも良い
- 深追いしない。月 1 回、気になった 1 件を学習会のきっかけにする使い方を推奨

### エビデンスの狙い

「退院日の決定は、ベースライン ADL 復帰ではなく退院後の予後を見据えた機能最適化で判断する」というパラダイムを、カンファの中で少しずつ浸透させるための仕組みです。押しつけは避け、数字で静かに示します。

## 第6章：カンファ終了・記録（2 分）

### 記録の仕組み

- ステータス変更・患者情報編集はすべて即時保存されているので、「保存を忘れた」は起きない
- ただしセッションが閉じるとブラウザ内のタブ状態は消えるので、**画面全体のスクリーンショットを撮影する** 運用にする
- スクリーンショットは病棟 PC のフォルダ（院内 LAN で師長・MSW と共有）に保存

### 推奨：スクリーンショット印刷

- ブラウザの「印刷」機能で PDF 化 → 院内プリンタで A3 横印刷
- 印刷時は自動的にファクトバーと編集ボタンが消えて、カンファ結果だけが紙に残る
- 医療記録として保存する場合は患者名・ID を伏せ字にするか、編集欄を空欄のまま運用する

### 次回カンファへの引き継ぎ

- ステータスは JSON 保存されているので、1 週間後に同じ画面を開くと **前回のステータスから再開できる**
- 新規入院が発生している場合は、サンプルデータ 10 名の枠内でカテゴリを 🆕 に戻す運用（Phase 2 で実データ接続予定）
- データ管理エクスパンダー「🗑 データ管理（一括クリア）」で、週初めに全員 🆕 へリセットする運用も可能（詳細は下記）

### 発言例（師長・カンファ閉会）

> 「今日のカンファはここまで。スクリーンショットを撮って 6F 師長にも共有します。来週の火曜日、同じ時間に再集合でお願いします」

## 補章 A：連休対策モードの運用タイミング

### 切替の仕組み — 手動切替 + 推奨バナー

- 連休対策モードは **師長が手動で ON** します（自動切替ではありません）
- 連休まで **21 日以下** になると、画面上部に推奨バナー「💡 連休まで N 日 — 連休対策モードへの切替を推奨します」が自動表示されます
- **14 日を切ると警告色（橙）、7 日以内は危険色（赤）** で強調表示されます
- バナーを見て判断し、**サイドバーの「連休対策モード」トグルを ON** にします
- 連休明け後、師長が手動で OFF に戻します

### 切替手順

1. 画面上部の推奨バナー（💡 連休まで N 日…）を確認
2. サイドバーの「連休対策モード」トグルを ON
3. 画面上の変化を確認：
   - モードバッジが「🚨 連休対策モード」赤に変わる
   - 推奨バナーは非表示になる（切替済みのため）
   - ブロック B が連休 5 日分の見通しに切り替わる
   - ブロック C のステータス候補が 7 種 → 4 種（✅確定 / 🛠調整中 / 🏥継続 / 🆕）に変わる
   - ブロック D の職種別お願いも連休向け文言に変わる

### 切替後の 3 週間の進め方

| 週 | 目標 |
|----|------|
| 連休 3 週前 | 対象患者の洗い出し、確定・調整中・継続の初回カテゴリ分け |
| 連休 2 週前 | 調整中患者 → 確定 or 継続への移行を進める |
| 連休 1 週前 | 確定患者の搬送・迎え時間の最終確認、継続患者の連休中指示書確認 |

### 運用ケース例

- **例1（14 日前）:** 「今朝確認したら画面上部に橙色のバナーが出ている。連休 14 日前で推奨が出たので、『連休対策モード』を ON にして画面を切り替える」
- **例2（7 日前）:** 「赤色のバナーに変わる。退院調整の最終段階に入ったサイン」

### 連休明けの再切替

- 連休明けの火曜カンファで、師長が **手動で** 通常モードに戻す（自動復帰はしません）
- データは保持されるが、カテゴリ体系が変わるので全患者が 🆕 新規 になる
- 連休前退院で不在になった患者は、次回カンファ時に枠外になる（サンプルデータ運用の仕様）

## 補章 B：患者名・患者 ID を編集する場合

### いつ編集するか

- カンファ参加者が患者をすぐ特定できない場合（「伊藤医師の Day 35 って誰だっけ？」）
- 実データ運用フェーズ（Phase 2）移行時
- スクリーンショットに残さない運用なら、カンファ中だけ一時入力して終了時にクリアする運用も可

### 操作

1. 患者行の右端「✏️」ボタン（ペンのアイコン）をクリック
2. ポップオーバーが開く
3. 「主治医名」「患者名」「患者 ID」の 3 つのテキストフィールド
4. 入力するとフォーカスを外した瞬間に保存される
5. 同じポップオーバー内「🗑 クリア」エクスパンダーで個別クリア可能

### 個人情報保護の運用ルール

- **Git に上がらない**：`data/patient_names.json` は `.gitignore` で除外済
- **ローカル保存のみ**：クラウド同期対象外、院内 PC にのみ残る
- **スクリーンショット運用**：患者名を入れるとスクリーンショットにも映る。院外共有する場合は事前に一括クリアを推奨
- **一括クリア**：ヘッダー直下「🗑 データ管理（一括クリア）」エクスパンダーで全患者一括クリア可能

### 一括クリアの手順

1. ヘッダー直下の「🗑 データ管理（一括クリア）」エクスパンダーを開く
2. クリア対象を 4 つのチェックボックスから選択（主治医名 / 患者名 / 患者 ID / ステータス）
3. 確認欄に「クリア」と入力
4. 実行ボタンが有効化されるのでクリック
5. 件数が表示されて完了

## 補章 C：カンファを回せるようになるまでの慣らし期間

- **初回**：司会がこの台本を読みながら進める。25〜30 分かかってよい
- **2〜3 回目**：参加者全員がステータスタグの意味を把握。20 分に短縮
- **4 回目以降**：タグクリック → 議論 → 自動更新の流れが体に入る。15〜20 分で完走
- **1 ヶ月後**：この台本を見なくても回せる状態がゴール

## よくあるつまずきと対処

| つまずき | 原因 | 対処 |
|---------|------|------|
| カンファ中にステータスタグが反応しない | ブラウザが固まっている | ページを再読み込み（ステータスは保存済） |
| 前回のステータスが残っていない | データ管理で一括クリアした後 | `data/patient_status.json` は最後の保存日時を確認 |
| 連休対策モードに切り替えたのに 7 タグのまま | モードトグル切替直後はリセットされる仕様 | 一度全患者 🆕 に戻り、そこから 4 タグに分類し直す |
| スクリーンショットに患者名が写る | 事前に名前を入力してあった | 印刷前に一括クリアを実行 |
| 画面が 16:9 に収まらない | ブラウザのズームが 100% 以外 | Ctrl/Cmd + 0 でリセット |
| エビデンスが毎回同じ | 乱数シードの問題 | アプリ再起動で別ファクトが出る |
