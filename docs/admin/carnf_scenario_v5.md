# 多職種カンファ v5 台本

- **作成日:** 2026-04-27
- **対象アプリ:** ベッドコントロールシミュレーター v5 試験運用版
- **対象:** 病棟師長、主任看護師、退院支援看護師、MSW、担当医、副院長
- **想定所要時間:** 通常 20〜25 分、連休対策時 30〜40 分
- **画面:** `📊 今日の運営` → 必要時 `🏥 退院調整` / `📈 過去1年分析` / サイドバー `🏥 疾患別マニュアル`

---

## v5 の目的

v5 では、機能説明ではなく **朝の判断順** に合わせてカンファを進めます。

1. まず小型 KPI で「今朝の状態」を即答する
2. 次に最上段の行動フォーカスカードで「今日あと何をするか」を 1 つに絞る
3. 詳細サマリーは必要な時だけ開く
4. 6F 看護必要度は、適応のある患者を疾患別マニュアルで確認する
5. 最後に試験運用アンケートへつなげる

この台本は、司会者だけが手元で見る想定です。参加者全員に読ませる資料ではありません。

---

## v5 で必ず守る運用原則

### 医療倫理ガードレール

- 不必要な処置を追加しない
- 評価表や記録を虚偽記載しない
- 病棟都合で患者を選別しない
- 看護必要度のために治療を変えるのではなく、**適応のある治療・ケア・記録の取りこぼしを防ぐ**

### 6F の基本方針

- 6F は入院数を減らす提案ではない
- 稼働率維持のため、入院数は過去実績ペースを前提にする
- 2026 新基準対策は、分母削減ではなく **適応のある患者日を見落とさない** 方向で進める
- 件数ペースは目標ではなく、適応評価後の参考値として扱う

## 第0章 開始前準備（3 分）

### 操作

1. ブラウザでアプリを開く
2. サイドバーで `📊 今日の運営` を選択
3. データが入っていることを確認
4. 必要なら `☀️ 本日の詳細サマリー` は閉じたままにする
5. サイドバーの `📚 看護必要度クイックリファレンス` が開けることを確認

### 司会の確認

- 小型 KPI strip が表示されているか
- 最上段に行動フォーカスカードが出ているか
- 稼働率、空床、在院患者数を口頭で言えるか
- 6F の看護必要度を議題にする日か

## 第2章 KPI strip と行動フォーカスカード（3 分）

### 見る場所

画面最上段。

1. 小型 KPI strip: 稼働率 / 空床 / 在院患者数
2. 行動フォーカスカード: 今日あと何をするか
3. 必要時のみ `☀️ 本日の詳細サマリー`

### 進め方

1. 司会が KPI strip を読み上げる
2. 行動カードの `title` を読む
3. `action` の冒頭にある安全前提を読む
4. chips の 3 つの数字だけ確認する
5. 詳細サマリーは、質問が出た時だけ開く

### 発言例

> 「今朝は 6F 稼働率 __%、空床 __床、在院患者 __名です」

> 「今日の一手は、画面のカードでは『________』です。まずこの 1 点に絞ります」

> 「ここに件数ペースが出ている場合も、現場へのノルマではありません。適応のある患者を確認した後の参考値です」

### ここで決めること

- 今日、誰に何を確認するか
- 退院調整を進めるのか
- 入院受入枠を作るのか
- 6F 必要度の確認を同時に行うのか

## 第3章 退院調整の本体（10〜15 分）

### 操作

1. サイドバー `🏥 退院調整` を開く
2. `🏥 カンファ資料` を確認
3. 必要なら病棟を 5F / 6F で切り替える
4. 連休前後は連休対策モードを確認する

### 見る順番

1. 退院日が確定している患者
2. 医学的には退院可能だが、家族・施設調整が残っている患者
3. 医学的安全性の確認が必要な患者
4. 週末・連休に退院が集中していないか
5. 退院後に入院受入枠として使える日がどこか

### 発言例

> 「まず、今日確定できる退院日から確認します。医学的に安全か、家族・施設調整が済むかを分けて見ます」

> 「退院を急がせるのではなく、安全に退院できる患者さんの調整を前に進めます」

> 「週末前に空床ができる場合は、連携室と外来に共有して、受入可能な患者さんがいないか確認します」

### 決定欄

| 決めること | 担当 | 期限 |
|---|---|---|
| 退院日を確定する患者 |  | 今日 __ 時 |
| 家族・施設へ確認する患者 |  | 今日 __ 時 |
| 主治医へ医学的安全性を確認する患者 |  | 今日 __ 時 |
| 連携室へ空床情報を共有する日 |  | 今日 __ 時 |

## 第4章 6F 看護必要度を同時に見る場合（5〜7 分）

### 使う場面

- 6F の必要度 II が基準未達または未達見込み
- 6F に内科・ペイン科の入院が多い
- 医師が「どの患者が該当しうるか」をまだ掴めていない
- 看護師から「この処置は評価に入るのか」と質問が出た

### 操作

1. サイドバー `📚 看護必要度クイックリファレンス` を開く
2. `🏥 疾患別マニュアルを開く` をクリック
3. 領域フィルタまたは検索で疾患を探す
4. まず `最初に見る` タブだけ読む
5. 必要時だけ `評価・記録`、`具体例`、`根拠・安全` を開く

### 司会の言い方

> 「ここは点数を取りに行く画面ではありません。目の前の患者さんに、適応のある治療・ケア・記録があるかを確認する画面です」

> 「まず疾患を検索して、“最初に見る”だけ読みます。詳しい根拠は必要になった時に見ます」

### 医師に依頼する確認

| 確認 | 例 |
|---|---|
| A 項目 | 酸素、注射薬剤 3 種類以上、シリンジポンプ、輸血、昇圧剤、抗血栓塞栓薬の持続点滴 |
| C 項目 | CV 挿入、腰椎穿刺、ERCP/ESD/内視鏡止血、PTCD、PEG、CART、消化管ステント |
| 記録 | 適応、開始日、終了予定、継続理由、切替予定 |
| 安全 | 適応外処置ではないか、治療選択として妥当か |

### 看護師に依頼する確認

| 確認 | 例 |
|---|---|
| 当日の状態 | 酸素使用、創傷処置、吸引以外の呼吸ケア、救急搬送後 |
| 処置の開始・終了 | 点滴、シリンジポンプ、輸血、ドレーン、CV |
| 医師への確認 | 翌日持ち越しにせず、当日中に確認 |
| 記録の同期 | 看護記録、処置記録、オーダーのズレ |

### 避ける言い方

- 評価目的に聞こえる言い方
- 処置追加を促すように聞こえる言い方
- 患者さんを制度達成の手段に見せる言い方
- 病棟運営を患者選択の理由に見せる言い方

### 推奨する言い方

- 「適応があるか確認する」
- 「記録の取りこぼしを確認する」
- 「医学的に妥当な選択肢を同日確認する」
- 「治療と記録を同期する」

## 第5章 What-if と過去1年分析への寄り道（3〜5 分）

### What-if・戦略を使う場面

- 行動カードの提案だけでは判断が決まらない
- 退院前倒し、入院受入、需要増減のどれが効くかを比較したい
- 会議で「何人動かせば変わるのか」と聞かれた

### 発言例

> 「What-if は患者選別や適応外対応のためではなく、実行可能な運用条件を 1 つだけ動かして、稼働率や空床がどう変わるかを見る画面です」

> 「今日は退院前倒し人数だけを動かします。複数の条件を同時に触ると、何が効いたか分からなくなります」

### 過去1年分析を使う場面

- 6F の月間入院数が過去実績どおりか確認したい
- ペイン科比率や内科比率を確認したい
- 看護必要度 II の不足患者日を説明したい
- 理事会・師長会議で背景資料にしたい

### 発言例

> 「過去1年分析は、毎朝見る画面ではありません。今日の判断が過去実績と比べて自然かどうかを確認するために使います」

## Appendix A スクリーンショット記録

カンファ議事録代わりに残す場合は、以下の 3 枚だけで十分です。

1. KPI strip + 行動フォーカスカード
2. 退院調整カンファ資料
3. 6F 必要度または疾患別マニュアルを開いた画面

個人情報が画面に含まれる場合は、院内 PC 内で保存し、Git や外部クラウドへ上げないでください。
