# ChatGPT Pro 用プロンプト: シナリオ vs アプリ整合性監査

以下のプロンプトをChatGPT Proに貼り付けて使用してください。
GitHubリポジトリに直接アクセスし、シナリオ台本とアプリコードを突き合わせて不整合を検出します。

## チェック観点

### 1. 数値の整合性
シナリオ台本に登場するすべての具体的な数値（稼働率、LOS、救急搬送比率、余力日数、床数、金額など）が、デモデータとアプリの計算ロジックから実際に算出される値と一致するか確認してください。

不一致の例:
- シナリオ「5Fが約17.7日、6Fが約21.3日」→ アプリの制度余力画面で表示される数値は同じか？
- シナリオ「稼働率91%」→ デモデータから計算すると実際は何%か？
- シナリオ「救急搬送比率22%/2.6%」→ admission_details.csvから計算すると一致するか？

### 2. 画面遷移・UI要素の整合性
シナリオ台本が参照している画面・ボタン・タブ・セクション名が、app.pyの実装と一致するか確認してください。

チェック項目:
- サイドバーのセクション名（📊ダッシュボード / 🎯意思決定支援 / 🛡️制度管理 等）
- タブ名（制度余力 / 需要波 / C群コントロール / 救急搬送15%）
- シナリオの「操作:」行で指示している画面遷移パスが実際のUIと一致するか
- シナリオで言及しているUI要素（結論カード、KPI信号、翌朝受入余力カード等）が実際に存在し、シナリオの説明通りに動作するか
- 結論カードに「対象: ○○病棟」または「対象: 病院全体」が表示されるか
- KPIストリップの救急搬送比率にモード名（院内運用用/届出確認用）が表示されるか

### 3. 用語の整合性
シナリオ台本の用語集に載っている用語と、アプリのUI上の表記が一致するか確認してください。
- 「営業日」と「診療日」の混在がないか
- 「制度ガードレール」と「施設基準チェック」の混在がないか
- その他、同じ概念に複数の表記が使われていないか

### 4. 機能の存在確認
シナリオ台本が説明している機能が、アプリに実際に実装されているか確認してください。
- シナリオで言及しているが実装されていない機能はないか
- 実装されているがシナリオで言及されていない重要な機能はないか

### 5. 表示条件・エッジケース
- 「データなし」「データ不足」と表示される条件を確認し、デモデータでその状態にならないか
- 病棟選択（全体/5F/6F）による表示切替が正しく動作するか
- 各KPIの閾値（5床以上=余裕、3-4床=要注意、2床以下=不足 等）がシナリオの説明と一致するか

### 6. テストカバレッジ
- 上記で発見した不整合が、既存テストで検出できる状態か
- `scripts/hooks/smoke_test.py` のシナリオ整合性チェックが、シナリオ台本の全数値をカバーしているか

- 推計値（proxy）は厳密一致でなく、妥当な範囲（±10%程度）であれば「一致」と判定してください

## 使い方

1. 上記の ``` で囲まれた部分をコピー
2. ChatGPT Pro に貼り付け
3. ChatGPT Pro がGitHubにアクセスしてコードを読み、対比表を生成
4. 生成された不整合リストをClaude Codeに渡して修正を指示

## 定期的な使い方

アプリを更新した後、このプロンプトを再実行することで、
シナリオとアプリの乖離を自動検出できます。
