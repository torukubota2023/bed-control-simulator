# Claude × Codex 協働の記憶

> 副院長（おもろまちメディカルセンター）が Claude Code と Codex を並走させて
> 意思決定する運用の蓄積。次回セッション開始時の文脈共有・引き継ぎ用。

## 使い方

- **次回 Claude/Codex セッション開始時**にこのファイルを共有する
- 新しい協働事例が出たら「§ 4 事例集」に追記
- 3 ヶ月ごとに「§ 5 学び」を見直し、陳腐化したものは削除

---

## 1. 役割と特徴

### Claude Code（私）
- **育つ土台**: ルール × 会話（外部脳 = `CLAUDE.md`、`memory/*.md`）
- **強み**: 思考整理、戦略の言語化、可視化、実装の早出し
- **癖**: データから推測しがち、積み上げ楽観、仕様書を後回しにする
- **記憶**: セッション間ゼロ。CLAUDE.md と memory ファイルに依存

### Codex
- **育つ土台**: 環境 × 成果物（外部脳 = 実ファイル群）
- **強み**: 公式資料の参照、統計的厳密性、実務単位への翻訳、冷静なブレーキ役
- **癖**: 規範に寄りすぎる、Claude より提案速度が遅い場合あり

### 副院長（人間）
- **育てる側**: 両者のルールを書き留め、環境を整え、差分を裁定する
- **これがないと両者とも空回りする**

---

## 2. 協働パターン（うまくいく順序）

```
[Claude] 早出し
   粗い数字・初期実装・可視化を素早く出す
        ↓
[Codex] 検証
   公式資料・統計理論で精度チェック、誤り指摘
        ↓
[Claude] 修正・再実装
   指摘を反映してコード・図を更新
        ↓
[副院長] 裁定
   両者の差分を見て最終判断
```

逆順は機能しない（Codex に最初から走らせると提案が遅れ、Claude を後回しにすると実装速度の利点が活きない）。

---

## 3. 役割分担マトリクス

| 場面 | 主担当 | 副担当 |
|---|---|---|
| 思考整理・戦略言語化・対話的判断 | **Claude** | Codex（検証） |
| 環境探索・実ファイル操作・成果物化 | **Codex** | Claude（指示書化） |
| 重要判断 | **副院長** | 両者並走 → 差分裁定 |
| ルール書き留め | **副院長** | Claude（提案） |
| 公式仕様の確認 | **Codex** | Claude（応用） |

## 4. 事例集

### Case 1: 2026-04 看護必要度推定（2026-04-30 〜 05-02）

**タスク**: DPC 支援ツールが使えない移行期間中に、月次基礎データから看護必要度Ⅰ・Ⅱ を推定。

**Claude の貢献**:
- Ridge 回帰 + Bootstrap 95%CI のパイプライン実装（純 numpy、約 1,600 行）
- 可視化 8 図（月次推移、CI 横バー、LOOCV 検証、手法比較）
- 副院長との対話的な戦略整理

**Codex の貢献**:
- H ファイル仕様の正解を厚労省資料から提示
- 統計的リーク（LOOCV）の指摘
- 実務単位（患者日/月）への翻訳
- 自然変動 σ を考慮した楽観バイアスの抑制

**Codex が指摘した Claude の誤り**:
1. **H ファイル評価者コードの誤解**: ASS0021 を必要度Ⅱ A 項目と推測 → 実は B 項目（食事摂取等）
2. **LOOCV リーク**: 全期間で特徴量選択 → fold 内選択が正解
3. **「分母完全一致」表現の誇張**: 実際は最大 8 患者日差 → 「ほぼ一致」が正確
4. **6F-Ⅱ 達成見込みの楽観**: σ=4.5pt の自然変動を考慮していなかった

**両者一致した結論**:
- 6F-Ⅰ 届出を第一候補（Ⅰ -2.9pt < Ⅱ -5.2pt のため）
- Ⅱ は警報指標として継続監視
- 戦略 2 本柱:
  1. A/C/EFコード/看護記録の突合監査（+30〜50 患者日/月目標）
  2. C 項目候補（CV/ERCP/PEG 等）の即日フラグ運用（月 10 件目標）
- 短手 3 振替は 5F-Ⅱ 副作用のため不採用
- 5月単月評価はノイズ大、3 ヶ月平均で判定

**成果物**:
- 実装: `[[scripts/nursing_estimation/]]`
- 文書: `[[docs/nursing_estimation/2026apr_estimation_method]]`
- 図: `data/nursing_estimation/figures/5_*.png` 〜 `8_*.png`
- 副院長から事務（諸見里様）への報告: 500 字本文 + 4 図 + 戦略表

## 5. 学び（次回 Claude セッションでの自戒）

### Claude 側で気をつけたいこと
- [ ] **仕様書を先に当たる** — データから推測する前に厚労省資料・公式仕様を確認
- [ ] **LOOCV の厳密性** — fold 内で特徴量選択を再実行（リーク防止）
- [ ] **積み上げ楽観への自戒** — 効果数値を足す前に自然変動 σ を計算
- [ ] **実務単位への翻訳** — pt だけでなく「患者日/月」「件数」で語る癖
- [ ] **Codex 並走前提** — 重要判断は必ず Codex の独立検証を仰ぐ

### Codex 側に期待すること
- 早期の素早いフィードバックは Claude の方が得意なので、Codex は「丁寧に検証して締める」役を意識
- 公式資料の引用は最初の方で出してもらえると、Claude の手戻りが減る

### 副院長として運用すべきこと
- セッション間のメモリ（CLAUDE.md、memory ファイル、このファイル）の継続的整備
- Claude と Codex の差分が出た時、どちらか片方の意見だけで決断しない
- 重要な制度ルール・運用ルールは CLAUDE.md にコミット（実装と乖離させない）

## 7. メモリの陳腐化対策

このドキュメントは状況変化と共に古くなる。以下のタイミングで見直す:

- **5月実測値が出た時** — 推定モデルの精度ズレを記録、§ 4 Case 1 を更新
- **戦略実装 1 ヶ月後** — 2 本柱の実績を追記

## 8. 三角形の構造

最後に、このすべての前提として：

> **Claude（規律で動く）／ Codex（環境で動く）／ 副院長（両者を育て、突き合わせ、裁定する）**

優劣ではなく、副院長が頂点にいる関係。両 AI とも、整える人がいて初めて機能する道具です。
