# 木曜ハドル／連休対策カンファ資料 — 設計仕様書

**作成日:** 2026-04-17
**対象:** ベッドコントロールシミュレーター新ビュー（案D拡張版）
**ファイル予定:** `scripts/views/conference_material_view.py`

---

## 1. 目的

木曜ハドル（週末退院調整）と連休対策カンファを、**1 画面で運営できる Streamlit ビュー**を構築する。病棟師長・主任・リハ PT/OT/ST・退院支援看護師が参加し、退院日調整を**「退院時機能の最適化」**パラダイムで進める。

## 2. 臨床哲学

**「退院日の決定は、ベースライン ADL 復帰ではなく、退院後の予後を見据えた機能最適化で判断する」**

- 稼働率維持は副次的な結果。主は患者の長期予後
- 「延長で稼働率を上げる」という順序の論理は使わない
- エビデンスは [bed_control_conference_facts_master.md](bed_control_conference_facts_master.md) を参照

## 3. 確定した設計判断

### 画面基本仕様
| 項目 | 決定 |
|------|------|
| 出力形式 | Streamlit 1 画面（スクロールなし）＋ スクリーンショット印刷対応 |
| 画面比 | **16:9**（通常プロジェクター） |
| モード切替（木曜／連休対策） | **手動切替** |
| 病棟表示 | **プルダウン手動切替**（5F / 6F） |
| 表示患者数上限 | **10 名** |

### 稼働率目標の設定（Q1 = 🅳）
- **月次変動 × 病棟別**
- テーブル構造: `settings/occupancy_target.yaml`
  ```yaml
  5F:
    default: 92
    2026-04: 88   # GW 月は低め
    2026-05: 88
    2026-08: 86   # お盆月
    2026-12: 86   # 年末月
  6F:
    default: 88
    2026-04: 84
    ...
  ```

### ALOS 目標の設定（Q2 = 🅲）
- **制度の rolling 3 ヶ月上限と連動** — 地域包括医療病棟の施設基準 21 日以内
- 6/1 以降は rolling 3 ヶ月判定に完全移行（[CLAUDE.md](../../CLAUDE.md) 参照）
- 警告閾値: 21 日の 95% = **19.95 日** に近づいたら要注意
- 病棟別ではなく**全体判定**（制度ルール通り）

### 大型連休の定義（Q3 = 🅱）
- **3 連休以上すべて**
- 対象: 成人の日 / 春分の日 / GW / 海の日 / 山の日 / 敬老の日 / 秋分の日 / 体育の日（スポーツの日）/ 文化の日 / 勤労感謝の日 / 年末年始 / お盆など
- 判定モジュール: `scripts/holiday_calendar.py`
- 土日 2 連休は通常業務として扱う（連休対策モードを発動しない）

### 連休対策モードの表示（Q4 = 🅰）
- **手動切替のみ**（師長がカンファ前に切り替え）
- サイドバーに切替トグル配置
- 連休までのカウントダウン表示は常時（連休対策モード発動とは別）

### ② 計算表現（Q5 = 🅱）
- **金額のみ表示、前提は裏に隠す**
- 例: 「田中氏（5F Day18、金曜退院予定）→ 月曜延長で **7 万円回避**」
- 前提（土日 2 床日 × 3.5 万円 / 床日）は `?` アイコンで hover／click 展開
- 詳細が見えるエクスパンダーは実装するが、デフォルトは閉じる

## 4. 画面構成（4 ブロック + ファクト）

### ブロック A: 今月の目標達成進捗（常時）
- 稼働率: 今月の実績 / 目標（病棟別）
- ALOS: rolling 3 ヶ月実績 / 制度上限 21 日
- 救急搬送後 15% 進捗（施設基準）
- 残り診療日と必要な床日
- 連休までのカウントダウン（近い時のみ強調）

### ブロック B: 今週の見通し
- 週末受入余力予測（5F / 6F 別、選択病棟のみ表示 — 床日単位で応需可能だった患者機会を可視化）
- 救急余力（金・土・日）
- 連休対策モード時は連休期間全体の見通しに切替

### ブロック C: 個別調整提案（最大 10 名）
- 前倒し候補 / 延長候補
- 提案ごとに床日寄与と回避額を明示（前提は隠す）
- 患者情報は医師名・病棟・Day 数・推奨アクションのみ（個人情報保護）

### ブロック D: 職種別 今週の協力ポイント
- 🏃 リハ
- 📋 退院支援 NS
- 👔 医師
- 🧑‍⚕️ 看護

### ファクトローテーション（画面最下部、控えめ表示）
- 1 行 40-80 字
- レイヤー×状況連動の重み付けで選定（[マスターファイル](bed_control_conference_facts_master.md) 参照）
- `@media print { display:none }` で印刷時非表示
- PMID / 引用は hover で展開

## 5. 既存モジュールとの連携

| 既存モジュール | 役割 |
|-----------------|------|
| `bed_data_manager.py` | 稼働率・ALOS・空床データの取得 |
| `emergency_ratio.py` | 救急搬送後 15% の進捗 |
| `holiday_strategy_view.py` | 連休対策データ（4 週カレンダー等）の再利用 |
| `action_recommendation.py` | 既存の「今日の一手」ロジックと整合 |

## 6. 新規モジュール

| 新規モジュール | 役割 |
|-----------------|------|
| `views/conference_material_view.py` | ビュー本体（4 ブロック） |
| `patient_suggestion_engine.py` | 在院日数×診療科×医師パターンから患者別提案生成 |
| `holiday_calendar.py` | 3 連休以上を判定するカレンダー |
| `fact_rotation.py` | ファクト状況連動選定 |
| `data/facts.yaml` | 25 件のマスターファクト（PMID 付き） |
| `settings/occupancy_target.yaml` | 病棟×月の目標値テーブル |
