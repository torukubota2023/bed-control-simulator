# ベッドコントロールシミュレーター v3.5 実演シナリオ台本

## 概要

**対象**: 看護師長・病棟スタッフ・医師・経営層
**想定場面**: 看護師長会議または経営会議で、副院長がアプリの使い方を実演する
**所要時間**: 約15分（シナリオ0〜7 + まとめ）
**前提**: デモデータ（2026年4月）がロード済み。ブラウザでアプリを開いた状態から開始

### デモデータの主な数値（参照用）

| 指標 | 値 |
|------|------|
| 全体稼働率 | 約88.8% |
| 5F稼働率 | 約86.5% |
| 6F稼働率 | 約91.1% |
| 平均在院日数（5F rolling） | 約17.7日 |
| 平均在院日数（6F rolling） | 約21.3日 |
| 金曜退院集中率 | 約31% |

## シナリオ0: 結論カード — 画面を開いたら最初にここを見る（1分）

### 操作
1. サイドバーで「📊 ダッシュボード」セクションを選択

### 話すこと

> 「毎朝このアプリを開くと、最初に見えるのがこの **結論カード** です」

- **結論カード（今日の一手）** がダッシュボード最上部に表示されている
  - 「カードの色で緊急度がわかります。赤なら今すぐ対応、黄色なら要注意、緑なら順調です」
  - 「👉 で始まる行が、今日やるべきアクションの提案です」
  - 「判定根拠も表示されるので、なぜその提案なのかも分かります」

> 「結論カードの下に、KPIが並んでいます」

- **KPI優先表示** — 6つのKPIが重要度順に並んでいる
  - 「左から順に重要です。救急搬送比率 → 稼働率 → 翌朝受入余力 → 平均在院日数（LOS） → C群の順で並んでいます」
  - 「赤や黄色の信号があれば、それが今日の課題です」

> 「この一画面で、今日何をすべきかが30秒で分かります」

## シナリオ1: 翌朝の受入余力 — 明日の朝、救急を受けられるか（1分）

### 操作
1. ダッシュボードのKPI表示の下を確認

### 話すこと

> 「KPIの下に、**翌診療日朝の受入余力** が表示されています」

- 翌朝受入余力・3診療日最小・退院予定・判定が4列で表示
  - 「翌朝受入余力は推計○床です。判定が🟢余裕ありなら、明日の救急搬送は問題なく受けられます」
  - 「3診療日最小も確認してください。ここが3床未満だと、近日中にベッドが逼迫する可能性があります」
  - 「退院予定○名と合わせて、今日のうちに調整が必要かどうか判断できます」

> 「この数字が見えるだけで、夜勤帯の看護師長への申し送りが変わります」

## シナリオ2: 稼働率の全体像を把握する（2分）

### 操作
1. 「📊 日次推移」タブをクリック

### 話すこと

> 「日次推移タブで、稼働率の動きを20日間分見てみましょう」

- 全体稼働率が約88.8%と表示されている
  - 「目標の90%にあと1.2ポイント足りません。稼働率1%は年間約1,200万円の差です」
- 5Fと6Fの差に注目
  - 「5Fは86.5%、6Fは91.1%。5Fが全体を引き下げています」
  - 「週末に稼働率が落ちる谷のパターンが見えますか？ これが金曜退院集中の影響です」

### 操作
2. 「🔄 フェーズ構成」タブをクリック

### 話すこと

- A群（急性期）/B群（回復期）/C群（退院準備期）の構成比
  - 「C群の割合がどのくらいか確認してください。C群は退院調整の余地がある患者群です」
  - 「C群は院内の運用ラベルで、在院15日目以降の患者を指します。制度上の公式区分ではありません」

## シナリオ3: 施設基準チェック — 制度要件を守れているか（2分）

### 操作
1. サイドバーで「🛡️ 制度管理」セクションをクリック
2. 「🛡️ 制度・需要・C群」タブ → 「制度余力」サブタブを表示

### 話すこと

> 「制度管理セクションでは、地域包括医療病棟の施設基準を満たしているかをチェックします」

- **総合判定** が信号で表示されている
  - 「🟠未完（データ不足）と出ている場合は、在宅復帰率やADL低下割合のデータがまだ入っていないということです」
  - 「安全と判定されるまでは油断しないでください」

- **平均在院日数（LOS）の余力** が表示されている
  - 「LOS（Length of Stay = 平均在院日数）の余力は○日です」
  - 「余力が2日未満になると黄色信号です。C群の退院を早める検討が必要になります」
  - 「制度上限は固定値ではなく、85歳以上の患者割合によって20〜24日に変動します。当院は約25%なので上限は21日が目安です」

> 「施設基準を割ると入院料が変更になり、大幅な減収になります。ここは毎日チェックしてください」

## シナリオ4: 救急搬送後患者割合 — 15%基準の病棟別管理（3分）

### 操作
1. 「🚑 救急搬送15%」サブタブをクリック
2. 表示モードは「両方表示（推奨）」のまま

### 話すこと

> 「施設基準で最も注意が必要なのが、救急搬送後患者割合15%です」

- **5Fと6Fの対比** を見せる
  - 「5Fは22.7%で🟢安全、6Fは5.0%で🔴危険域です」
  - 「病院全体の平均だけ見ていると、6Fの問題を見落とします。病棟別に毎月チェックすることが重要です」

- **届出確認用と院内運用用の違い**
  - 「院内運用用に切り替えると、短手3（ポリペクなど）を分母から除きます。救急対応とは無関係な入院を除くことで、実態に近い数字が見えます」

### 操作
3. 月末着地予測を見せる

- 「6Fは標準シナリオでも○%で未達見込みです」

### 操作
4. 「あと何件必要か」を見せる

- 「6Fはあと○件の救急搬送が必要。残り診療日あたり○件/日です」

### 操作
5. 緊急アラート（出ている場合）を見せる

- 「赤いアラートが出ている場合は、救急搬送・下り搬送の受入を最優先にする運用を検討してください」
- 「専門外や満床の例外はありますが、基本方針として救急車を断らない意識を全病棟で共有しましょう」

## シナリオ5: 需要波 — 入院需要は増えているか減っているか（1分）

### 操作
1. 「🌊 需要波」サブタブをクリック

### 話すこと

> 「需要波タブでは、入院需要のトレンドを自動判定しています」

- **全体のトレンド** — 📈増加 / ➡️横ばい / 📉減少
  - 「前2週間と直近1週間の平均入院数を比較して判定しています」
- **経路別需要トレンド**（表示されている場合）
  - 「救急が減少傾向なら15%基準に注意。予定入院が増えていればベッド配分を考えましょう」

> 「繁忙期ならC群退院を早めて受入枠を確保。閑散期ならC群をキープして稼働率を維持。この判断の根拠がここにあります」

## シナリオ6: C群コントロール — 退院タイミングの調整余地（2分）

### 操作
1. 「📋 C群コントロール」サブタブをクリック

### 話すこと

> 「C群コントロールは、退院準備期の患者の退院タイミングを調整する際の判断材料を提供します」

- **C群アラート** に救急搬送比率のリスクが反映されている
  - 「C群を長くキープすると稼働率は維持できますが、ベッドが埋まって救急を受けられなくなるリスクがあります」
  - 「6Fは救急搬送比率が未達リスクなので、C群退院を優先してベッドを空けるべきです」

- **C群候補一覧（lite版）** を見せる
  - 「推定在院日数○日以上のC群候補が○名います」
  - 「これはあくまで推計です。延長推奨リストではありません。臨床判断・家族の状況・退院支援体制を踏まえて、最終判断は担当医が行ってください」

- **トレードオフの考え方**
  - 「C群をキープするか退院させるかは、制度要件・稼働率・受入余力の3つのバランスで決まります」
  - 「ダッシュボードの結論カードでも、このトレードオフの結論が出ています」

## まとめ（1分）

> 「このアプリの使い方をまとめます」

1. **毎朝30秒**: ダッシュボードの結論カードとKPIを確認 → 今日の優先アクションを把握
2. **週1回**: 制度管理セクションで施設基準チェック → 救急搬送比率15%・LOS余力を確認
3. **月1回**: What-if分析で改善仮説を検討 → 経営会議への提案材料に

> 「大事なのは、精神論ではなく数字で判断することです。"ベッドを空けないようにしましょう"ではなく、"今日は受入余力が○床あるから、救急搬送はあと○件受けられる"——そういう会話ができるようになることが、このアプリの目標です」

## 補足: 用語メモ

| 用語 | 意味 |
|------|------|
| **LOS** | Length of Stay（平均在院日数）。入院してから退院するまでの日数の平均 |
| **C群** | 在院15日目以降の患者（院内運用ラベル。制度上の公式区分ではない） |
| **施設基準チェック** | 地域包括医療病棟の施設基準を満たしているかの確認 |
| **短手3** | 短期滞在手術等基本料3。ポリペク等の短期入院で、LOS計算から除外される |
| **proxy / 推計** | 統計的に推定した値。実測値とは異なる場合がある |
| **稼働率** | （在院患者数＋退院患者数）÷ 病床数。厚労省定義に準拠 |
| **下り搬送** | 他院から救急患者連携搬送料を算定して搬送された患者 |
