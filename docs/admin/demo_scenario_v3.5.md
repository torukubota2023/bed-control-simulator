# ベッドコントロール支援アプリ — 教育用シナリオ台本 v3.5

---

## A. 設計方針

1. **一本の問い**で通す。「6Fで今日は何を優先するか」を冒頭に提示し、シナリオ全体をその解決過程として構成する。機能紹介ツアーにしない。
2. **判断の流れ**で章立てする。画面遷移順ではなく、「受入余力を見る → 原因を分解する → トレードオフを比較する → 制度要件を確認する → アクションを決める」の順。
3. **対象は臨床現場**。看護師・医師が日常業務で使う言葉を優先する。収益・経営指標の話は appendix に分離する。
4. **推計値の限界**を繰り返し明記する。このアプリは判断を補助するツールであり、入院予測や退院指示を自動で決めるものではない。最終判断は担当医・病棟チームが行う。

### 導入：6Fで今日は何を優先するか（1分）

> 「これから紹介するのは、病床の運営判断を支援するアプリです。機能の説明ではなく、**実際の判断の流れ**に沿って使い方を見ていきます」

> 「今日の問題設定はこれです」

**問い：6Fで今日、退院調整と救急受入のどちらを優先すべきか？**

> 「6Fは稼働率91%で高めに見えます。一方で、救急搬送比率が15%を下回るリスクがあります。退院を遅らせて稼働率を維持するのか、退院を進めてベッドを確保し救急を受けるのか。この判断をデータで支えるのがこのアプリの目的です」

> 「では画面を見ていきましょう」

---

### 第1章：今朝の受入余力を確認する（2分）

**画面：📊 ダッシュボード**

> 「毎朝アプリを開くと、最初に見えるのがこの画面です。順に見ていきます」

#### 結論カード

- 画面最上部に色つきのカードが表示されている
  - 「このカードが **今日いちばん大事なこと** を1行で伝えています」
  - 「色は緊急度です。赤なら今すぐ対応、黄色なら要注意、緑なら順調」
  - 「👉 で始まる行が、具体的なアクション提案です」

#### 状況の信号

- カードの下に、救急搬送比率・施設基準チェック・稼働率・受入余力・平均在院日数（LOS）・C群の状況が信号つきで並んでいる
  - 「赤や黄色の信号があるところが、今日の課題です」

#### 翌診療日朝の受入余力

- 翌朝受入余力・3診療日最小・退院予定・判定の4列
  - 「翌朝受入余力が○床、判定は🟢余裕あり（または🟡要注意）と出ています」
  - 「この数字は過去7日の曜日パターンから推計した値です。退院予定や予定入院の変動により実際とは異なる場合があります」
  - 「5床以上なら余裕、3〜4床なら要注意、2床以下なら不足です」

> 「ここまでで30秒です。この習慣をつけるだけで、朝の判断の精度が変わります」

---

### 第2章：余力が不足する原因を分解する（3分）

> 「受入余力の数字だけでは、何が原因で余裕があるのか／ないのかが分かりません。原因を分解していきましょう」

#### 2-1. 平均在院日数（LOS）の状況

**画面：🛡️ 制度管理 → 🛡️ 制度・需要・C群 → 制度余力**

**操作**: サイドバーで「🛡️ 制度管理」をクリック → 「制度余力」サブタブ

- 平均在院日数の余力が表示されている
  - 「LOS（平均在院日数）は、入院してから退院するまでの日数の平均です」
  - 「デモデータでは、5Fが約17.7日、6Fが約21.3日です。6Fは制度上限の21日にほぼ到達しています」
  - 「LOS余力が2日未満になると黄色信号です。6Fは余力がほとんどありません」

> 「つまり6Fは、患者の退院を遅らせる余地がほとんどない状態です」

#### 2-2. 入院需要のトレンド

**操作**: 「🌊 需要波」サブタブをクリック

- 全体のトレンド（📈増加 / ➡️横ばい / 📉減少）が表示されている
  - 「前2週間と直近1週間の平均入院数を比べて判定しています」
  - 「需要が増えているなら受入に備える必要がありますし、減っているなら空床対策を考えます」

> 「受入余力の裏側にある "LOS余力" と "需要の波" を確認しました。次に、退院調整と受入のバランスをどう取るかを見ます」

---

### 第3章：退院調整と受入のトレードオフを比較する（3分）

**操作**: 「📋 C群コントロール」サブタブをクリック

> 「ここが今日の問いの核心です」

#### C群とは

- 「まず用語の確認です。C群とは **在院15日目以降の患者** を指す、当院の運用ラベルです。制度上の公式区分ではありません」
- 「C群の患者は、退院タイミングの調整余地が比較的大きいため、このアプリでは調整候補として表示しています」
- 「ただし、**延長推奨リストではありません**。退院の判断は臨床状態・家族の状況・退院支援体制を踏まえて、担当医と病棟チームが行います」

#### 6Fのトレードオフ

- C群アラートに救急搬送比率のリスクが反映されている
  - 「C群を長くキープすると稼働率は維持できます。しかしベッドが埋まることで、救急搬送を受けられなくなります」
  - 「6Fは救急搬送比率が15%を下回るリスクがあります。アプリもC群退院を推奨しています」

- C群候補一覧（表示されている場合）
  - 「推計在院日数や退院近接度から候補を一覧表示しています。すべて推計値です」
  - 「ここから退院調整会議の議題を拾うイメージで使ってください」

#### トレードオフ評価カード

- ダッシュボードにも表示されていたトレードオフ評価
  - 「C群をキープした場合と退院させた場合で、受入余力・LOS余力・救急搬送比率がどう変わるかを比較しています」

> 「6Fの答えが見えてきました。LOS余力がなく、救急搬送比率も未達リスクがある以上、**C群退院を進めてベッドを確保し、救急受入を優先する**のが今日の方針です」

---

### 第4章：制度要件の安全確認（2分）

> 「方針が決まったら、その判断が制度要件に照らして安全かどうかを最終確認します」

#### 4-1. 施設基準チェック

**画面：🛡️ 制度管理 → 制度余力**

- 総合判定を確認
  - 「🟢安全 / 🟡注意 / 🔴危険 / 🟠未完（データ不足）の4段階です」
  - 「未完と出ている場合は、在宅復帰率やADL低下割合などのデータが入力されていません。安全と判定されるまでは安心しないでください」

- 制度上限の仕組み
  - 「平均在院日数の上限は固定値ではありません。85歳以上の患者割合によって20日〜24日に変動します」
  - 「当院は85歳以上が約25%なので、上限は約21日が目安です」

#### 4-2. 救急搬送後患者割合15%

**操作**: 「🚑 救急搬送15%」サブタブをクリック

- 5Fと6Fの対比
  - 「5Fは約22.7%で🟢安全。6Fは約5.0%で🔴危険域です」
  - 「全体の平均だけでは6Fの問題を見落とします。**病棟ごとに毎月チェック**することが重要です」

- 「あと何件必要か」の表示
  - 「6Fは15%達成にあと○件の救急搬送が必要。残り診療日あたり○件/日です」

- 届出確認用と院内運用用
  - 「短手3（ポリペクなど）を分母から除くと、実態に近い数字が見えます。画面上部のモード切替で連動して変わります」

> 「施設基準を割ると入院料が変更になり、大幅な減収につながります。第3章で決めた方針——C群退院を進めて救急受入を優先する——は、制度面からも正しい判断です」

---

### 第5章：朝会・日中・週1回レビューでの使い分け（2分）

> 「最後に、このアプリをいつ・どう使うかを整理します」

#### 毎朝の確認（30秒）

1. ダッシュボードを開く
2. 結論カードの色と提案を確認 → 今日の最優先アクション
3. 翌朝受入余力の数字を確認 → 救急受入に支障がないか

> 「この3ステップが習慣になるだけで、朝の申し送りが具体的になります。"今日は受入余力○床なので、救急搬送は受けられる状態です" と言えるようになります」

#### 日中の判断サポート

- 退院調整会議の前にC群候補一覧を確認
- 救急搬送の受入可否判断に受入余力を参照
- 「繰り返しますが、アプリの数字は判断の補助です。患者の状態や家族の希望は画面に映りません。最終判断は現場のチームが行ってください」

#### 週1回のレビュー

- 制度管理セクションで施設基準チェック・救急搬送比率15%を確認
- 需要波で入院トレンドの変化を確認
- 看護師長会議やカンファレンスの資料として活用

---

### 第6章：まとめ（1分）

> 「今日の問い——6Fで退院調整と救急受入のどちらを優先するか——に対して、アプリの画面を使って以下の手順で答えを出しました」

1. 受入余力を確認した
2. LOS余力と需要トレンドで原因を分解した
3. C群退院と救急受入のトレードオフを数字で比較した
4. 制度要件に照らして判断の安全を確認した

> 「"ベッドが空いているから入院を取ろう" "忙しいから退院を早めよう" ではなく、受入余力・制度余力・退院調整のバランスを数字で見て判断する。この考え方を日常に取り入れることが、このアプリの目標です」

> 「最終判断をするのは、画面の前の皆さんです。アプリはそのための材料を提供します」

## 用語集

| 用語 | 意味 |
|------|------|
| **LOS** | Length of Stay（平均在院日数）。入院から退院までの日数の平均 |
| **C群** | 在院15日目以降の患者。当院の運用ラベルであり、制度上の公式区分ではない |
| **施設基準チェック** | 地域包括医療病棟の施設基準を満たしているかの確認（旧称：制度ガードレール） |
| **受入余力** | 翌診療日の朝に救急搬送を受け入れ可能な推計床数 |
| **短手3** | 短期滞在手術等基本料3。ポリペク等の短期入院。LOS計算からは除外される |
| **推計（proxy）** | 統計的に推定した値。実測値とは異なる場合がある |
| **稼働率** | 当日の在院患者数に退院患者数を加えた値を病床数で割ったもの |
| **需要波** | 入院需要のトレンド（増加・横ばい・減少）を前2週間と直近1週間の比較で判定したもの |
| **下り搬送** | 他院から救急患者連携搬送料を算定して搬送された患者。救急搬送比率の分子に含まれる |

## Appendix：経営層向けの補足

以下の内容は、経営会議や理事会での報告に使う場合の補足です。臨床現場向けの主台本とは分離しています。

### A-1. 稼働率と収益の関係

- 当院（94床）では、稼働率1%の変動は年間約1,200万円の診療報酬収入差に相当する
- デモデータの全体稼働率は約88.8%。目標の90%まであと1.2ポイント
- サイドバーに常時表示される「稼働率1% ≒ 年間約1,200万円」はこの計算に基づく

### A-2. What-if分析の活用

**画面：🎯 意思決定支援 → 🔮 What-if分析**

- 退院の前倒しや充填確率をスライダーで動かし、改善効果をシミュレーションできる
- 名前をつけてシナリオを保存し、複数のシナリオを比較検討できる
- 経営会議への提案材料として活用

### A-3. 運営改善アラート

**画面：🎯 意思決定支援 → 🚨 運営改善アラート**

- 金曜退院集中（約31%）、週末空床パターンなどの改善ポイントが自動検出される
- 空床コストが金額換算で表示されるため、改善の優先順位がつけやすい

### A-4. C群の運営貢献額

- C群患者1名・1日あたりの運営貢献額は約28,900円（推計）
- 空床にすると貢献額は0円。C群をキープすることは稼働率の下支えになる
- ただし制度要件（LOS上限・救急搬送比率）を超えない範囲でのみ有効

### A-5. HOPE送信連携

**画面：📨 HOPE連携 → 📨 HOPE送信**

- 富士通HOPE電子カルテのToDo機能で、病床状況のサマリーメッセージを生成できる
- 制度アラート・救急搬送比率・C群状況を統合した400文字以内のメッセージ
- コピーしてHOPEのToDoに貼り付けるだけで、全スタッフへの情報共有が完了

### A-6. 2026年度診療報酬改定の影響

- 2024年度比で日次運営貢献額が約20万円/日増加（推計）
- 年間換算で約7,700万円の増収効果
- 施設基準の維持がこの増収の前提条件であり、制度管理の重要性がさらに高まっている

## E. ファイル構成案

本ファイル（`docs/admin/demo_scenario_v3.5.md`）を正本とする。

```
docs/admin/
├── demo_scenario_v3.5.md          ← 正本（本ファイル）
├── demo_scenario_v3.3.md          ← 旧版（参考保存）
```
