# ベッドコントロール支援アプリ v3.6 — 臨床現場向けシナリオ台本

**対応アプリバージョン:** v3.6
**対象:** 病棟師長・主任看護師・医師・副院長（臨床運営の実務担当）
**想定所要時間:** 14 分（5 章構成 + 用語集 + Appendix）
**想定場面:** 院内勉強会・新任ベッドコントロール担当への導入説明・部署別ブリーフィング

---

## A. 設計方針

1. **一本の問い**で通す。「6F で今日、何を優先するか」を冒頭で提示し、シナリオ全体をその解決過程として構成する。機能一覧の紹介ツアーにしない
2. **判断の流れ**で章立てする。今日の運営確認 → 制度と需要の健全性 → 多職種カンファ → 連休前対応 → 仮説検証、の順で、サイドバー 5 セクションを一筆書きで通る
3. **対象は臨床現場**。看護師・医師が日常業務で使う言葉を優先する。経営用語は Appendix に分離する
4. **推計値の限界**を繰り返し明記する。このアプリは判断を補助するツールであり、入院予測や退院指示を自動で決めるものではない。最終判断は担当医・病棟チームが行う

## C. サイドバー 5 セクション構成の概観（事前共有）

このアプリのサイドバーは 5 セクションに再編されています：

| セクション | 役割 | 含むタブ |
|-----------|------|---------|
| 📊 今日の運営 | 朝の総合状況確認 | 意思決定ダッシュボード / 運営改善アラート / 日次推移 / フェーズ構成 / 運営分析 / トレンド分析 |
| 🔮 What-if・戦略 | 仮説検証・シミュレーション | What-if 分析 / 仮説管理 |
| 🛡️ 制度管理 | 施設基準・需要・C 群 | 制度・需要・C 群 / 改善のヒント |
| 🏥 退院調整 | 多職種カンファと連休対策 | 🏥 カンファ資料 / 退院タイミング / 今週の需要予測 / 退院候補リスト / 予約可能枠 |
| ⚙️ データ・設定 | 運用管理・マスター | 日次データ入力 / 実績分析・予測 / 医師別分析 / 医師マスター / データエクスポート / HOPE 送信 / 短手3 設定 |

朝の運営では「📊 今日の運営」→「🛡️ 制度管理」で状況確認、午前の多職種カンファでは「🏥 退院調整」で議論、必要に応じて「🔮 What-if・戦略」で仮説検証という動線です。

### 第1章：朝の確認 — 📊 今日の運営（2 分）

**操作：** サイドバーで「📊 今日の運営」をクリック → 「🎯 意思決定ダッシュボード」タブが選択されている

> 「毎朝アプリを開くと、最初に見るのがこの画面です。順に見ていきます」

#### 結論カード（画面最上部）

- 画面最上部に色つきのカードが表示されている
  - 「このカードが **今日いちばん大事なこと** を 1 行で伝えています」
  - 「色は緊急度です。赤なら今すぐ対応、黄色なら要注意、緑なら順調」
  - 「👉 で始まる行が、具体的なアクション提案です」

#### 状況の信号

- カードの下に、救急搬送比率・施設基準チェック・稼働率・受入余力・平均在院日数（LOS）・C 群の状況が信号つきで並ぶ
  - 「赤や黄色の信号があるところが、今日の課題です」

#### 翌診療日朝の受入余力

- 翌朝受入余力・3 診療日最小・退院予定・判定の 4 列
  - 「翌朝受入余力が ○床、判定は🟢余裕あり（または🟡要注意）と出ています」
  - 「この数字は過去 7 日の曜日パターンから推計した値です。退院予定や予定入院の変動により実際とは異なる場合があります」
  - 「5 床以上なら余裕、3〜4 床なら要注意、2 床以下なら不足です」

- **3 診療日最小の意味**
  - 「翌朝の空床数だけを見ていると、数日後にベッドが急減するリスクを見落とします」
  - 「3 診療日最小は、**向こう 3 診療日の中で最もベッドが少なくなる日の予測値** です」
  - 「例えば、翌朝 10 床なのに 3 日最小が 3 床だったら、明後日以降に急激に詰まる予兆です。今日のうちに退院調整を前倒しすべきです」
  - 「つまり **翌朝は今日の判断、3 日最小は先手を打つための判断材料** です」

- 病棟別の受入余力
  - 「その下に 5F・6F の病棟別受入余力が表示されています」
  - 「救急搬送は入院先の病棟に空床がなければ受けられません。**病院全体ではなく、病棟単位で確認する** ことが重要です」

> 「ここまでで 30 秒です。この習慣をつけるだけで、朝の判断の精度が変わります」

#### 【補足】運営改善アラートタブ

- **操作：** 2 番目のタブ「🚨 運営改善アラート」をクリック
- 「金曜退院集中」「週末受入余力の使い残し」など、改善のタネが自動検出されて一覧表示される
  - 「これは毎朝見るというより、週 1 回レビューのときに確認するタブです」

### 第2章：制度と需要の健全性 — 🛡️ 制度管理（3 分）

**操作：** サイドバーで「🛡️ 制度管理」をクリック → 「🛡️ 制度・需要・C 群」タブ内に 4 つのサブタブ（制度余力・需要波・C 群コントロール・救急搬送 15%）

> 「受入余力の数字だけでは、何が原因で余裕があるのか／ないのかが分かりません。原因を分解していきましょう」

#### 2-1. 制度余力サブタブ — LOS と施設基準

**操作：** 「🛡️ 制度余力」サブタブ

- 平均在院日数の余力が表示されている
  - 「LOS は、入院してから退院するまでの日数の平均です」
  - 「デモデータでは、5F が約 17.7 日、6F が約 21.3 日です。6F は制度上限の 21 日を **わずかに超過** しています」
  - 「LOS 余力が 2 日未満になると黄色信号、0 日以下は赤信号です。6F は余力が **-0.3 日（制度上限超過）** です」

- 施設基準チェック総合判定
  - 「🟢安全 / 🟡注意 / 🔴危険 / 🟠未完（データ不足）の 4 段階です」
  - 「未完と出ている場合は、在宅復帰率や ADL 低下割合などのデータが入力されていません。安全と判定されるまでは安心しないでください」

- 制度上限の仕組み
  - 「平均在院日数の上限は固定値ではありません。85 歳以上の患者割合によって 20 日〜24 日に変動します」
  - 「当院は 85 歳以上が約 25% なので、上限は約 21 日が目安です」

> 「6F は制度上限を超過しており、**退院調整を最優先で進める** 必要があります」

#### 2-2. 需要波サブタブ — 入院トレンド

**操作：** 「🌊 需要波」サブタブ

- 全体のトレンド（📈増加 / ➡️横ばい / 📉減少）が表示されている
  - 「前 2 週間と直近 1 週間の平均入院数を比べて判定しています」
  - 「需要が増えているなら受入に備える必要がありますし、減っているなら空床対策を考えます」

#### 2-3. C 群コントロールサブタブ — トレードオフの核心

**操作：** 「📋 C 群コントロール」サブタブ

- 「まず用語の確認です。C 群とは **在院 15 日目以降の患者** を指す、**院内運用ラベル** です。制度上の公式区分ではありません」
- 「C 群の患者は退院タイミングの調整がしやすいため、このアプリでは調整候補として表示しています。ただし **延長推奨リストではありません**。退院の判断は臨床状態・家族の状況・退院支援体制を踏まえて、担当医と病棟チームが行います」

- C 群候補一覧
  - 「C 群候補の平均在院日数が 21 日以下になるまで、在院日数の長い順に何人退院させればよいかを計算し、🔴急ぎ退院必要・🟡退院必要・🟢まだ在留可能の 3 段階で判定しています。すべて推計値です」
  - 「ここから退院調整カンファの議題を拾うイメージで使ってください」

#### 2-4. 救急搬送 15% サブタブ — 基準達成の確認

**操作：** 「🚑 救急搬送 15%」サブタブ

- 5F と 6F の対比
  - 「5F は約 22% で🟢安全。6F は約 2.6% で🔴危険域です」
  - 「全体の平均だけでは 6F の問題を見落とします。**病棟ごとに毎月チェック** することが重要です」

- 届出確認用と院内運用用の 2 系統表示
  - 「短手 3（ポリペクなど）を分母から除くと、実態に近い数字が見えます。画面上部のモード切替で連動して変わります」

> 「施設基準を割ると入院料が変更になり、大幅な減収につながります。6F は C 群退院を進めて救急受入を優先する、が今日の方針です」

> 「ここまでで、数字ベースの方針は決まりました。次は、多職種で合意形成する段階に入ります」

### 第3章：多職種退院調整カンファ — 🏥 退院調整 → カンファ資料（3 分）

**操作：** サイドバーで「🏥 退院調整」をクリック → 左端の「🏥 カンファ資料」タブが選択される

> 「ここが v3.6 の目玉機能です。病棟詰め所の PC で、師長・主任・リハ・退院支援 NS・担当医が画面を囲んで使う想定の 1 画面カンファビューです」

#### 画面上部：病棟とモードの切替

- ヘッダー直下に「病棟」プルダウンと「連休対策モード」トグル
  - 「今日は 5F のカンファとして話を進めます。連休対策モードは師長が手動で ON にする運用で、画面上部の推奨バナー（💡 連休まで N 日 …）が出たら切り替える流れです」

#### ブロック A：今月の目標達成進捗（5 つの KPI 横並び）

- 稼働率 85% / 目標 92% / -7pt
- 平均在院日数 17.5 日 / 上限 21 日まで余裕
- 救急搬送後 17% / 基準達成
- 残り診療日 9 日 / 必要床日 +28
- 次の大型連休：○日

> 「今月の進捗が一目で分かります。稼働率がマイナス 7 ポイント、ALOS は余裕あり、救急達成。**無理に延ばさずに、新しい患者を受ける戦略で動ける状況** ということが読み取れます」

#### ブロック B：今週の見通し + 在院患者ステータス内訳

- 週末 3 日分（金・土・日）の空床予測・救急余力
- 「週末受入余力 XX 床日（応需可能だった患者機会）」

> 「応需可能だった患者機会、という言葉を使っています。医療倫理上、空床を『損失』『コスト』と呼ばずに『受け入れられた可能性のある患者数』として見る、という当院の運用です」

#### ブロック C：個別患者のステータス分類（ここが議論の中心）

- 最大 10 名の患者行、在院日数の降順
- 各行にクリッカブルな **ステータスタグ**
  - ⚫ 方向性未決 / 🔵 医学的 OK 待ち / 🟢 家族希望待ち / 🟡 施設待ち / 🟣 介護保険待ち / 🟠 リハ最適化中 / 🆕 新規 の 7 種
- タグをクリックするとポップオーバー → ラジオボタンで選択 → 即座に保存 + 内訳が自動更新される

> 「カンファの流れはこうです。師長が司会で、患者を 1 名ずつ読み上げます。MSW・リハ・主治医が発言し、師長が PC で該当タグをクリックしてカテゴリを確定させます。10 名を 10 分で回せる設計です」

> 「カテゴリ分けが進むにつれて、左の内訳数字が刻々と更新されます。最終的に『方向性未決 3 名、施設待ち 2 名』のように状態が一目で分かる」

- 右端の ✏️ ボタンで主治医名・患者名・患者 ID を編集可能（任意、空欄で運用可）
  - 「個人情報の運用：入力データは `data/patient_names.json` に保存され、`.gitignore` で Git 追跡禁止にしてあります。スクリーンショット印刷時は事前に一括クリアする運用を推奨します」

#### ブロック D：職種別の今週のお願い（4 列）

- リハ / 退院支援 NS / 医師 / 看護の 4 役割
- ブロック C のカテゴリ分類に応じて内容が自動変化

> 「ブロック C でステータス分類が変わるたびに、このブロック D の依頼事項が連動して更新されます。師長がカンファ後にタスクを洗い出す作業が要らなくなる設計です」

#### ファクトバー（画面最下部・紺色）

- 「退院時 IC が 1 点高いごとに 1 年死亡 OR 0.98」等のエビデンスが 1 行で表示
- 毎回ローテーション、印刷時は自動非表示

> 「このエビデンスバーは、『退院日の決定は、ベースライン ADL 復帰ではなく退院後の予後を見据えた機能最適化で判断する』というパラダイムを、カンファの中で少しずつ共有するための仕組みです。15 秒で読めるようにしています」

> 「カンファの詳細運用は別途 [carnf_scenario_v1.md](carnf_scenario_v1.md) を参照してください」

### 第4章：連休前対応 — 🏥 退院調整 → 連休対策モード + 今週の需要予測（3 分）

> 「大型連休は需要の谷です。連休前に退院を進めないと連休明けに病床が詰まり、救急を受けられなくなります。このセクションで対応します」

#### 4-1. カンファ資料を連休対策モードに切替

**切替の仕組み（2026-04-18 確定運用）:** 連休対策モードは **師長が手動で ON** します（自動切替ではありません）。ただし、連休まで **21 日以下** になると画面上部に推奨バナー「💡 連休まで N 日 — 連休対策モードへの切替を推奨します」が自動表示されるため、師長はそれを合図に切替判断を行います。14 日を切ると警告色（橙）、7 日以内は危険色（赤）で強調表示されます。

**操作：** 推奨バナーを確認した上で、サイドバーの「連休対策モード」トグルを ON

- モードバッジが「🚨 連休対策モード」赤に変わる
- 推奨バナーは非表示になる（切替済みのため）
- ブロック B が連休 5 日分の見通しに切り替わる（例：4/29 水〜5/3 日）
- ブロック C のステータスが ✅連休前退院 確定 / 🛠連休前退院 調整中 / 🏥連休中継続ケア / 🆕新規 の 4 種に変わる
- ブロック D の職種別お願いも連休向け文言に変わる

> 「今朝、画面上部に橙色の推奨バナーが出ていました。連休まで 14 日を切ったサインです。このカンファが終わったら、サイドバーのトグルを ON にして連休対策モードへ切り替えます。切替後の 3 週間の進め方は、対象患者の洗い出し → 調整中→確定への移行 → 搬送・迎え時間の最終確認、という流れです」

#### 4-2. 今週の需要予測タブ

**操作：** 「🏥 退院調整」セクション内の「📊 今週の需要予測」タブをクリック

- 病棟別の今週の入院予測件数・曜日別分布
  - 「予定入院の予約状況と過去の曜日パターンから、今週の入院件数を予測しています」
  - 「連休前の週は予測が急増する傾向。ここの数字を見ながらブロック B の空床見込みと突き合わせます」

#### 4-3. 退院候補リストタブ

**操作：** 「📋 退院候補リスト」タブをクリック

- 在院日数・経過の観点から退院調整の議題になりうる候補を抽出
  - 「ブロック C の個別議論と異なり、こちらは機械的に抽出されたロングリスト。カンファ前に目を通しておく用途です」

#### 4-4. 予約可能枠タブ

**操作：** 「📅 予約可能枠」タブをクリック

- 今週〜来週の日別予約可能枠（空床予測 + 退院予定）
  - 「連携室や外来で予定入院を入れる際、どの日なら枠があるかを確認するためのタブです」
  - 「連休明け初日の予約を連休前に埋めすぎないよう、ここで枠配分を設計する使い方」

> 「連休対策は『事前に退院枠を確保し、予約枠を data-driven に最適化する』の 2 本立て。カンファ資料 + 今週の需要予測 + 予約可能枠、この 3 タブを行き来しながら議論します」

### 第5章：仮説検証 — 🔮 What-if・戦略（2 分）

**操作：** サイドバーで「🔮 What-if・戦略」をクリック → 「🔮 What-if 分析」タブが選択される

> 「方針が決まったら、数字で裏付けを取ります」

#### What-if 分析

- 退院の前倒し人数（スライダー）× 充填確率（スライダー）
  - 「『木曜に 3 人前倒し退院・充填確率 80%』というシナリオを入力すると、稼働率・運営貢献額がリアルタイムで変わります」
  - 「感覚の議論を数字の議論に変えるための道具です」

#### 仮説管理

**操作：** 「💾 仮説管理」タブ

- 現在のスライダー設定に名前をつけて保存
- 複数の仮説を比較検討できる
  - 「カンファで議論した方針を仮説として保存しておき、1 週間後に実績と突き合わせる。これを繰り返すと、自院の予測精度が上がります」

> 「仮説検証のタブは、副院長・師長レベルで使う場面が多く、カンファの時間内では頻繁には使いません。カンファ後の振り返りや、経営会議への提案材料を作る時に使います」

---

### 締め：朝会・カンファ・週 1 レビューの使い分け（1 分）

| 時間帯 | 使うセクション | 何を見る |
|--------|-------------|---------|
| 毎朝 30 秒 | 📊 今日の運営 → 意思決定ダッシュボード | 結論カードと翌朝受入余力 |
| 多職種カンファ 20 分 | 🏥 退院調整 → カンファ資料 | 10 名のステータス分類・職種別お願い |
| 週 1 回レビュー 15 分 | 🛡️ 制度管理 + 📊 運営改善アラート | 施設基準・救急 15%・改善のタネ |
| 連休推奨バナー表示時（21 日以下） | 🏥 退院調整 → 連休対策モードを手動 ON | 連休前退院枠確保 |
| 月末・経営会議前 | 🔮 What-if・戦略 | 仮説検証・シナリオ比較 |

> 「今日の問い——6F で退院調整と救急受入のどちらを優先するか——に対して、アプリを使って以下の手順で答えを出しました」

1. 受入余力を確認した（今日の運営）
2. LOS 余力・施設基準・救急 15% で原因を分解した（制度管理）
3. 多職種カンファで個別患者の方向性を合意した（退院調整 → カンファ資料）
4. 連休前対応の準備を確認した（退院調整 → 連休対策モード）
5. 数字で裏付けを取った（What-if・戦略）

> 「『ベッドが空いているから入院を取ろう』『忙しいから退院を早めよう』ではなく、受入余力・制度余力・退院調整のバランスを数字で見て判断する。この考え方を日常に取り入れることが、このアプリの目標です」

> 「最終判断をするのは、画面の前の皆さんです。アプリはそのための材料を提供します」

## 用語集

| 用語 | 意味 |
|------|------|
| **LOS** | Length of Stay（平均在院日数）。入院から退院までの日数の平均 |
| **C 群** | 在院 15 日目以降の患者。院内運用ラベルであり、制度上の公式区分ではない |
| **施設基準チェック** | 地域包括医療病棟の施設基準を満たしているかの確認 |
| **受入余力** | 翌診療日の朝に救急搬送を受け入れ可能な推計床数 |
| **3 診療日最小** | 向こう 3 診療日の中で最も空床が少なくなる日の予測値。先手を打つための判断材料 |
| **短手 3** | 短期滞在手術等基本料 3。ポリペク等の短期入院。LOS 計算や救急搬送分母の扱いに特殊ルールがある |
| **推計（proxy）** | 統計的に推定した値。実測値とは異なる場合がある |
| **稼働率** | 当日の在院患者数に退院患者数を加えた値を病床数で割ったもの |
| **需要波** | 入院需要のトレンド（増加・横ばい・減少）を前 2 週間と直近 1 週間の比較で判定したもの |
| **下り搬送** | 他院から救急患者連携搬送料を算定して搬送された患者。救急搬送比率の分子に含まれる |
| **届出確認用（official）** | 短手 3 を含む救急搬送後患者割合。厚生局への届出確認に使用 |
| **院内運用用（operational）** | 短手 3 を除外した救急搬送後患者割合。日常の運用判断に使用 |
| **週末受入余力** | 週末または連休期間の空床を床日で合計し、応需可能だった患者機会として可視化する指標 |
| **応需可能だった患者機会** | 空床を金額で語らず、受け入れられた可能性のある患者数として可視化する表現 |
| **ステータスタグ** | カンファ資料のブロック C で患者ごとに付与するカテゴリ。通常 7 種、連休対策 4 種 |
| **連休対策モード** | 大型連休前の運用モード。師長が手動で ON/OFF する。カンファ資料のブロック B・C・D が連休向けに切り替わる。連休まで 21 日以下になると画面上部に推奨バナーが表示され、それを合図に ON にする流れ |
| **連休対策モード推奨バナー** | 通常モード時、連休まで 21 日以下で画面上部に自動表示される「💡 連休まで N 日 — 連休対策モードへの切替を推奨します」。8-21 日は警告色（橙）、7 日以内は危険色（赤）。モード ON で非表示、OFF で再表示 |
| **ファクトバー** | カンファ資料画面の最下部に表示されるエビデンス 1 行。印刷時は自動非表示 |
| **残り診療日** | 今月の残り日数のうち、診療を行う営業日数 |

## Appendix：経営層向けの補足

以下の内容は、経営会議や理事会での報告に使う場合の補足です。臨床現場向けの主台本とは分離しています。

### A-1. 稼働率と収益の関係

- 当院（94 床）では、稼働率 1% の変動は年間約 1,046 万円の診療報酬収入差に相当する
- デモデータの全体稼働率は約 88.8%。目標の 90% まであと 1.2 ポイント
- サイドバーに常時表示される「稼働率 1% ≒ 年間約 1,046 万円」はこの計算に基づく

### A-2. What-if 分析の活用

**画面：🔮 What-if・戦略 → 🔮 What-if 分析**

- 退院の前倒し人数と充填確率をスライダーで動かし、改善効果をシミュレーションできる
- 名前をつけてシナリオを保存し、複数のシナリオを比較検討できる
- 経営会議への提案材料として活用

### A-3. 運営改善アラート

**画面：📊 今日の運営 → 🚨 運営改善アラート**

- 金曜退院集中（約 31%）、週末受入余力の使い残しなどの改善ポイントが自動検出される
- 床日換算と年換算の両方で表示され、改善の優先順位がつけやすい

### A-4. C 群の運営貢献

- C 群患者 1 名・1 日あたりの運営貢献は約 28,900 円（推計）
- 空床にすると貢献は 0 円。C 群をキープすることは稼働率の下支えになる
- ただし制度要件（LOS 上限・救急搬送比率）を超えない範囲でのみ有効

### A-5. HOPE 送信連携

**画面：⚙️ データ・設定 → 📨 HOPE 送信**

- 富士通 HOPE 電子カルテの ToDo 機能で、病床状況のサマリーメッセージを生成できる
- 制度アラート・救急搬送比率・C 群状況を統合した 400 文字以内のメッセージ
- コピーして HOPE の ToDo に貼り付けるだけで、全スタッフへの情報共有が完了

### A-6. 2026 年度診療報酬改定の影響

- 2024 年度比で日次運営貢献額が約 20 万円／日増加（推計）
- 年間換算で約 7,700 万円の増収効果
- 施設基準の維持がこの増収の前提条件であり、制度管理の重要性がさらに高まっている
- 2026-06-01 以降は経過措置終了。救急搬送 15% は rolling 3 ヶ月判定、LOS は短手 3 階段関数対応が本則適用
