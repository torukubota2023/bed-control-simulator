# ベッドコントロールシミュレーター v4 — 臨床現場向けシナリオ台本

**対応アプリバージョン:** v4（2026-04-19 時点）
**対象:** 病棟師長・主任看護師・医師・副院長（臨床運営の実務担当）
**想定所要時間:** 約 18 分（導入 + 第1〜6章 + 用語集 + Appendix）
**想定場面:** 院内勉強会・新任ベッドコントロール担当への導入説明・部署別ブリーフィング
**アプリ URL:** https://bed-control-simulator.streamlit.app

---

## A. 設計方針

1. **一本の問い**で通す。「6F で今日、何を優先するか」を冒頭で提示し、シナリオ全体をその解決過程として構成する。機能一覧の紹介ツアーにしない
2. **判断の流れ**で章立てする。今日の運営確認 → 過去実績の文脈で見直す → 制度と需要の健全性 → 多職種カンファ → 連休前対応 → 仮説検証、の順で、サイドバー 5 セクションを一筆書きで通る
3. **対象は臨床現場**。看護師・医師が日常業務で使う言葉を優先する。経営用語は Appendix に分離する
4. **推計値の限界**を繰り返し明記する。このアプリは判断を補助するツールであり、入院予測や退院指示を自動で決めるものではない。最終判断は担当医・病棟チームが行う
5. **v4 で追加された 5 機能**（🅰〜🅴）は、各章の必要な場面で自然に触れる。新機能の紹介ではなく、判断の流れを補強する道具として位置づける

## C. サイドバー 5 セクション構成の概観（事前共有）

このアプリのサイドバーは 5 セクションに再編されています：

| セクション | 役割 | 含むタブ |
|-----------|------|---------|
| 📊 今日の運営 | 朝の総合状況確認・過去実績の俯瞰 | 意思決定ダッシュボード / 運営改善アラート / 日次推移 / フェーズ構成 / 運営分析 / トレンド分析 / **📉 過去実績分析（v4 新規）** |
| 🔮 What-if・戦略 | 仮説検証・シミュレーション | What-if 分析 / 仮説管理 |
| 🛡️ 制度管理 | 施設基準・需要・C 群 | 制度・需要・C 群 / 改善のヒント |
| 🏥 退院調整 | 多職種カンファと連休対策 | 🏥 カンファ資料 / 退院タイミング / 今週の需要予測 / 退院候補リスト / 予約可能枠 |
| ⚙️ データ・設定 | 運用管理・マスター | 日次データ入力 / 実績分析・予測 / 医師別分析 / 医師マスター / データエクスポート / HOPE 送信 / 短手3 設定 |

朝の運営では「📊 今日の運営」で状況把握 → 「🛡️ 制度管理」で原因分解、午前の多職種カンファでは「🏥 退院調整」で議論、月次レビューや経営報告では「📉 過去実績分析」を開く、という動線です。

**病床情報（毎回の前提）**

- 総病床数 **94床**（5F 47床 [外科・整形] / 6F 47床 [内科・ペイン]）
- 月間入院 約 **150 件**
- 稼働率目標 **全病棟 全月 一律 90%**（95% は最大限）
- **稼働率 1% の価値: 年間 1,199 万円**（常勤医師 1 名の手取り年収相当）

### 第1章：朝の確認 — 📊 今日の運営（2 分）

**操作：** サイドバーで「📊 今日の運営」をクリック → 「🎯 意思決定ダッシュボード」タブが選択されている

> 「毎朝アプリを開くと、最初に見るのがこの画面です。順に見ていきます」

#### 画面で見るもの

- **サイドバー最上部：経過措置終了までのカウントダウン**（🅰）
  - 「🗓️ 経過措置終了まで あと **XX 日**」のバナーが表示されている

- **結論カード（画面最上部）**

  - 赤なら今すぐ対応、黄色なら要注意、緑なら順調
  - 「👉」で始まる行が具体的なアクション提案

- **状況の信号（結論カードの下）**
  - 救急搬送比率・施設基準チェック・稼働率・受入余力・平均在院日数・C 群の状況が信号つきで並ぶ

### 第5章：連休前 + 連休中の分散退院 — 🏥 退院調整 → 連休対策モード + 今週の需要予測（3 分）

#### v4 での運用思想転換（副院長 2026-04-19 指示）

従来の連休前集中型には 3 つの課題がありました:
1. 連休前数日に退院が集中し、病棟スタッフの負荷が偏る
2. 連休中の稼働率が 60% 台まで落ち込み、診療報酬が大きく低下する <!-- qa-skip -->（従来型の問題イメージ）
3. 家族が平日に仕事を休んでお迎えに来る必要がある

v4 ではこれを **分散型** に変えます。3 つの柱で支えます:

1. **連休中退院の推奨化** — 従来の「日曜退院推奨」ロジックを連休中にも拡張
2. **一日 N 名までの退院枠** — 連休 5 日間で合計 M 名を分散（一日に集中させない）
3. **休日の家族都合を逆に利用** — 平日お迎えより休日お迎えの方が家族にも優しい

稼働率推移のイメージ:

| 時期 | 従来型（集中） | v4 新型（分散） |
|------|:---:|:---:|
| 連休前 | 95% | 90% |
| 連休中 | **60%** | **85%** |
| 連休後 | 80% | 85% |
| 全期平均 | 78% | **87%** |

> 「連休中の稼働率を 10〜20pt 改善できると、稼働率 1% = 年間 1,199 万円の係数から、**連休 1 回あたり数百万円規模の診療報酬改善**になります」

## 用語集

| 用語 | 意味 |
|------|------|
| **LOS** | Length of Stay（平均在院日数）。入院から退院までの日数の平均 |
| **C 群** | 在院 15 日目以降の患者。院内運用ラベルであり、制度上の公式区分ではない |
| **施設基準チェック** | 地域包括医療病棟の施設基準を満たしているかの確認 |
| **受入余力** | 翌診療日の朝に救急搬送を受け入れ可能な推計床数 |
| **3 診療日最小** | 向こう 3 診療日の中で最も空床が少なくなる日の予測値。先手を打つための判断材料 |
| **短手 3** | 短期滞在手術等基本料 3。ポリペク等の短期入院。LOS 計算や救急搬送分母の扱いに特殊ルールがある |
| **短手 3 階段関数**（2026-06-01 以降） | Day 5 まで LOS 分母から除外、**Day 6 以降に滞在が延びた瞬間に入院初日まで遡って全日数を分母にカウント**。Day 5/6 境界で LOS 分母が +6 日 jump する不連続点が発生する |
| **rolling 3 ヶ月判定**（2026-06-01 以降） | 救急搬送後患者割合 15% の判定を、単月ではなく 3 ヶ月合算で行う本則ルール。病棟別（5F / 6F 単体）で判定 |
| **85 歳以上緩和** | 85 歳以上患者割合が 20% 以上の月は LOS 上限が +1 日緩和。2026-06-01 以降は 20 日 → **21 日**（22 日ではない）。当院は通年 27.3% で条件クリア |
| **経過措置**（〜2026-05-31） | 令和 6 改定の経過措置期間。現行ルール（LOS 21 日 / 緩和 22 日、救急単月判定）が適用。2026-06-01 翌日から本則完全適用 |
| **推計（proxy）** | 統計的に推定した値。実測値とは異なる場合がある |
| **稼働率** | 当日の在院患者数に退院患者数を加えた値を病床数で割ったもの |
| **需要波** | 入院需要のトレンド（増加・横ばい・減少）を前 2 週間と直近 1 週間の比較で判定したもの |
| **下り搬送** | 他院から救急患者連携搬送料を算定して搬送された患者。救急搬送比率の分子に含まれる |
| **届出確認用（official）** | 短手 3 を含む救急搬送後患者割合。厚生局への届出確認に使用 |
| **院内運用用（operational）** | 短手 3 を除外した救急搬送後患者割合。日常の運用判断に使用 |
| **週末受入余力** | 週末または連休期間の空床を床日で合計し、応需可能だった患者機会として可視化する指標 |
| **応需可能だった患者機会** | 空床を金額で語らず、受け入れられた可能性のある患者数として可視化する表現 |
| **ステータスタグ** | カンファ資料のブロック C で患者ごとに付与するカテゴリ。通常 7 種、連休対策 4 種 |
| **連休対策モード** | 大型連休前の運用モード。師長が手動で ON/OFF する。カンファ資料のブロック B・C・D が連休向けに切り替わる。連休まで 21 日以下になると画面上部に推奨バナーが表示され、それを合図に ON にする流れ |
| **連休対策モード推奨バナー** | 通常モード時、連休まで 21 日以下で画面上部に自動表示される「💡 連休まで N 日 — 連休対策モードへの切替を推奨します」。8-21 日は警告色（橙）、7 日以内は危険色（赤）。モード ON で非表示、OFF で再表示 |
| **ファクトバー** | カンファ資料画面の最下部に表示されるエビデンス 1 行。PubMed MCP で一次文献検証済み（80 件、DOI 整合率 100%）。印刷時は自動非表示 |
| **残り診療日** | 今月の残り日数のうち、診療を行う営業日数 |
| **過去実績分析タブ**（v4 新規） | 📊 今日の運営 セクションの末尾タブ。FY2025 の 1,965 件の入院データから、サマリー KPI / 月別推移 / 曜日パターン / 時間帯 / 年齢分布 / 予約リードタイムの 6 セクションを表示 |
| **予約リードタイム** | 外来で入院予約を入れてから実際に入院するまでの日数。当院中央値 12 日 / 平均 16.3 日 |
| **実データ由来デモ**（v4 新規） | サイドバーの「📊 実データ由来デモを読み込む（2025FY）」ボタンで、2025 年度 365 日分（1,965 件）の実データベースのデモを一括ロードする機能 |
| **需要予測モデル校正**（v4） | 2025FY 実データを用いた需要予測モデルのチューニング。日別 MAE 12.7% 改善、日曜 44% 改善。裏側で自動適用 |

## Appendix：経営層向けの補足

以下の内容は、経営会議や理事会での報告に使う場合の補足です。臨床現場向けの主台本とは分離しています。

### A-1. 稼働率と診療報酬収入の関係

- 当院（94 床）では、**稼働率 1% の変動は年間約 1,199 万円の診療報酬収入差** に相当する（常勤医師 1 名の手取り年収相当）
- デモデータの稼働率はサマリー画面で **88.8%**、5F 約 85%、6F 約 91%（いずれも目標 90% に対する位置づけ）
- サイドバーに常時表示される「稼働率 1% ≒ 年間約 1,199 万円」はこの計算に基づく
- 稼働率目標は **全病棟 全月 一律 90%**、95% は最大限の水準

### A-3. What-if 分析の活用

**画面：🔮 What-if・戦略 → 🔮 What-if 分析**

- 退院の前倒し人数と充填確率をスライダーで動かし、改善効果をシミュレーションできる
- 名前をつけてシナリオを保存し、複数のシナリオを比較検討できる
- 経営会議への提案材料として活用

### A-4. 運営改善アラート

**画面：📊 今日の運営 → 🚨 運営改善アラート**

- 金曜退院集中、週末受入余力の使い残しなどの改善ポイントが自動検出される
- 床日換算と年換算の両方で表示され、改善の優先順位がつけやすい

### A-5. C 群の運営貢献

- C 群患者 1 名・1 日あたりの運営貢献は約 28,900 円（推計）
- 空床にすると貢献は 0 円。C 群をキープすることは稼働率の下支えになる
- ただし制度要件（LOS 上限・救急搬送比率）を超えない範囲でのみ有効

### A-6. HOPE 送信連携

**画面：⚙️ データ・設定 → 📨 HOPE 送信**

- 富士通 HOPE 電子カルテの ToDo 機能で、病床状況のサマリーメッセージを生成できる
- 制度アラート・救急搬送比率・C 群状況を統合した 400 文字以内のメッセージ
- コピーして HOPE の ToDo に貼り付けるだけで、全スタッフへの情報共有が完了

### A-7. 2026 年度診療報酬改定 + 2026-06-01 本則適用の影響

- 2024 年度比で日次運営貢献額が約 20 万円／日増加（推計）
- 年間換算で約 7,700 万円の増収効果
- 施設基準の維持がこの増収の前提条件であり、制度管理の重要性がさらに高まっている
- **2026-06-01 以降は経過措置終了**:
  - 救急搬送 15% は rolling 3 ヶ月判定（病棟別、短手 3 含む）
  - 平均在院日数は 20 日（85 歳以上 20% 以上の緩和で 21 日） <!-- qa-skip -->（制度上限の説明）
  - LOS 計算に短手 3 階段関数（Day 5 まで分母除外 → Day 6 以降全日数計上）
- 当院は FY2025 実績から **いずれの指標も基準を大幅クリア**。施設基準実態レポート（A-2）参照

### A-8. 需要予測モデル校正の成果（v4 新規）

- 2025FY 実データ（1,965 件）を用いたモデルのチューニングを v4 でリリース
- **日別予測 MAE 12.7% 改善**、**日曜予測誤差 44% 改善**
- 裏側で自動適用されており、ユーザーに見える設定項目はない
- 連休前の入院予測精度が改善 → 連休対策モードでの退院枠確保の精度が向上
