# 院内LAN展開計画 — ベッドコントロールシミュレーター

## 1. 想定構成

```
サーバーPC（Python + Streamlit）
    │
    │  院内LAN（有線/無線）
    │
    ├── 5F ナースステーション端末（ブラウザ）
    ├── 6F ナースステーション端末（ブラウザ）
    ├── 医局端末（ブラウザ）
    └── 事務室端末（ブラウザ）
```

- サーバーPC 1台で Streamlit アプリを起動（`--server.address 0.0.0.0 --server.port 8501`）
- 各病棟端末からブラウザで `http://[サーバーIP]:8501` にアクセス
- インターネット接続は不要（院内LAN完結）

## 2. 技術要件

| 項目 | 要件 |
|------|------|
| Python | 3.11 以上 |
| OS（サーバー） | Windows 10/11（院内標準PC） |
| ネットワーク | 院内LAN接続（ポート8501開放） |
| ブラウザ（端末） | 後述（Edge 90制約あり） |
| ストレージ | 500MB以上の空き容量 |

## 3. Edge 90 制約の説明

### 背景
- 院内標準ブラウザは **Microsoft Edge バージョン90**（Chromium 90、2021年4月リリース）
- 電子カルテシステムが Edge の特定バージョンに依存しており、**アップデートすると電子カルテが動作しなくなる**
- そのため Edge のバージョンは固定されており、変更不可

### 問題
- Streamlit 1.28+ のフロントエンドは **Chromium 90 では動作しない可能性が高い**
- Streamlit は React ベースのフロントエンドを使用しており、最新の JavaScript API（`Array.at()`、`Object.hasOwn()` 等）を要求する
- これはアプリコードの問題ではなく、**Streamlit フレームワーク自体のブラウザ互換性の問題**

## 4. 対応案の比較

### 案A: 古い Streamlit バージョンに固定（1.12〜1.16）

| 項目 | 評価 |
|------|------|
| 概要 | Edge 90 で動作する可能性のある古い Streamlit を使用 |
| メリット | 追加ソフト不要 |
| デメリット | 現行コードの `st.data_editor`（Streamlit 1.23+）等が使えない |
| デメリット | セキュリティパッチが提供されていない |
| デメリット | コードの書き換えが必要 |
| 工数 | 1〜2週間（コード修正＋動作検証） |
| リスク | 動作保証なし、将来の機能追加が困難 |

### 案B: ポータブルブラウザ併用（推奨）

| 項目 | 評価 |
|------|------|
| 概要 | PortableApps 版 Firefox をシミュレーター専用ブラウザとして配布 |
| メリット | **最新の Streamlit をそのまま利用可能** |
| メリット | **電子カルテへの影響ゼロ**（インストール不要、レジストリ変更なし） |
| メリット | Edge は電子カルテ専用、Firefox はシミュレーター専用で使い分け |
| デメリット | 各端末にポータブル Firefox を配置する手間 |
| 工数 | 1〜2日（配布＋ショートカット作成） |
| リスク | 低（実績のあるソリューション） |

### 案C: Flask 等での書き換え（最後の手段）

| 項目 | 評価 |
|------|------|
| 概要 | Streamlit を使わず、Flask + 素の HTML/CSS/JS で全 UI を再実装 |
| メリット | ブラウザ互換性を完全に制御可能 |
| デメリット | **工数 2〜3ヶ月**（全 UI の再実装） |
| デメリット | Streamlit の豊富なウィジェットを手動で再実装 |
| 工数 | 2〜3ヶ月 |
| リスク | 高（再実装によるバグ混入リスク） |

### 推奨: 案B（ポータブルブラウザ併用）

**案B を推奨する理由:**
1. 現行コードを一切変更せずに展開可能
2. 電子カルテへの影響が完全にゼロ
3. 導入工数が最小（1〜2日）
4. 将来の Streamlit アップデートにも追従可能

## 5. SE 確認事項チェックリスト

導入前に SE（システムエンジニア / IT部門）へ確認すべき事項:

- [ ] 院内LANでポート 8501 の通信は許可されているか
- [ ] サーバーPCの固定IPアドレスは取得可能か
- [ ] ファイアウォール / プロキシでローカル通信がブロックされていないか
- [ ] 共有フォルダ（`\\server\shared\` 等）へのファイル配置は可能か
- [ ] ポータブルアプリ（Firefox Portable）の端末への配置は許可されるか
- [ ] サーバーPCの常時起動（またはスケジュール起動）は可能か
- [ ] 端末のセキュリティソフトがポータブル Firefox の実行をブロックしないか

## 6. 導入前の必須検証 3 項目

### 検証1: ネットワーク疎通確認
- サーバーPCで Streamlit を起動
- 別の端末から `http://[サーバーIP]:8501` にアクセス
- ページが表示されることを確認

### 検証2: ブラウザ互換性確認
- `tools/browser_probe.html` を端末の Edge 90 で開き、JavaScript API の対応状況を確認
- ポータブル Firefox で同じページを開き、全項目がパスすることを確認
- Streamlit アプリの全画面（5セクション）が正常に表示されることを確認

### 検証3: 実運用シミュレーション
- 複数端末から同時アクセスし、レスポンス速度を確認
- パスワード認証が各端末で正常に動作することを確認
- データ入力 → 保存 → 再読み込みの一連の流れを確認

## 7. 導入後の確認 3 項目

### 確認1: 日常運用の安定性
- [ ] サーバーPCの再起動後、自動でアプリが起動するか（または手動起動手順が周知されているか）
- [ ] 1週間の連続運用でメモリリーク等の問題が発生しないか

### 確認2: ユーザー操作の定着
- [ ] 各病棟スタッフがポータブル Firefox のショートカットからアクセスできるか
- [ ] パスワード入力 → ダッシュボード表示の手順が理解されているか

### 確認3: データバックアップ
- [ ] CSV データファイルのバックアップ方法が決まっているか
- [ ] バックアップの頻度と保存先が設定されているか
