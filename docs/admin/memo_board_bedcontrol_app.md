# 院内メモ

**件名:** ベッドコントロール支援アプリ導入に関する提案（PoC実施依頼）

**宛先:** 理事長、理事会各位、各部門長

**差出人:** 副院長　久保田 徹

**日付:** 2026年3月31日

---

## 1. 背景と課題

当院（総病床数94床〔5F:47床、6F:47床〕、月間入院約150件、診療科構成：内科約50%・ペイン約15%・整形外科約15%・外科等）における病棟稼働率管理は、以前医師主導で退院日決定から退院調整依頼という形に変更となりましたが、現状ベッドコントローラーの経験と感覚に大きく依存しており、入退院判断の属人化による未活用病床コストが懸念されます。加えて、2026年度診療報酬改定に伴う地域包括医療病棟入院料1の算定要件（平均在院日数20日以内などのさまざまな施設基準）への適切な対応には、稼働率・在院日数・患者構成（A/B/C群）をリアルタイムで可視化し、データに基づく意思決定を行う体制の構築が急務です。

## 2. 提案内容

- **ベッドコントロール支援アプリ（制作維持コストなし）のPoC（実証運用）導入**（期間：2ヶ月間）
- **対象:** まず1病棟（6F：内科・ペイン）から開始
- **プラットフォーム:** 本アプリはStreamlit対応の軽量Webアプリケーションであり、院内LANに接続したサーバー1台（既存PCで可）に導入し、各端末のブラウザからアクセスする構成のため、外部へのデータ送信は発生せず、新規端末の導入も不要です。
- **主な機能:** 稼働率・患者群構成・運営貢献額のリアルタイム可視化、What-if分析（入退院シナリオ・退院計画・入院需要変動）、意思決定ダッシュボード（30秒朝アセスメント用スコアリング）、経済フラグ・アクション推奨

## 3. 期待される効果

- **空床の影響の可視化:** 空床1床あたり約2.5万円/日の未活用病床コストを数値化 ※1
- **稼働率改善による運営貢献額増:** 稼働率1%改善で **約70.5万円/月（年間約846万円）** の運営貢献額増（94床 x 1% = 0.94床 x 25,000円 x 30日）※1
- **稼働率目標レンジ（90-95%）の安定維持** による計画的ベッドコントロールの実現
- **意思決定の迅速化・標準化:** 朝会での病棟状態把握を30秒に短縮

## 4. PoC計画概要

| フェーズ | 期間 | 内容 |
|---------|------|------|
| Phase 1 | 1ヶ月目（4月前半） | デモモードで操作習熟・運用フロー確認 |
| Phase 2 | 月半ば以降（4月後半） | 6F病棟で実データ入力開始・日次運用 |
| Phase 3 | 2ヶ月目（5月） | 効果測定・全棟展開の可否判断 |

**PoC成功判定基準（KPI）:**
- 6F病棟の月間平均稼働率が **PoC開始前比で1%以上改善**（ベースラインは前年4月実績）
- ベッドコントローラーの **日次利用率80%以上**（稼働日のうちアプリを使用した日の割合）
- 運用担当者の **満足度スコア3.5/5.0以上**（操作性・有用性の5段階評価）

## 5. お願い事項

以下について、ご承認を賜りたく存じます。

1. 本PoCの実施承認
2. 看護部への協力依頼の裁可（ベッドコントローラー担当者の選定、日次運用への参画）

追加コストは発生せず、既存業務への負荷も最小限に設計しております。PoC終了後、効果が確認されれば5F病棟への展開を検討し、効果が不十分な場合は運用を終了いたします。

※1 本試算は運営貢献額ベース（変動費のみ差引き）。看護人件費等の固定費は空床でも発生するため、空床の真の未活用病床コストとして運営貢献額を採用。
