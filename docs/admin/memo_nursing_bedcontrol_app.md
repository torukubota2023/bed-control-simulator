## 1. 趣旨

理事会承認を前提として、病棟稼働率管理の可視化および意思決定支援を目的としたベッドコントロール支援アプリのPoC運用（3ヶ月間）を開始したく、看護部のご協力をお願い申し上げます。

本アプリは2026年度診療報酬改定（地域包括医療病棟入院料1、平均在院日数20日以内等の施設基準）のコストモデルに基づき、稼働率・患者構成・運営収支をリアルタイムで可視化するものです。まず6F病棟（内科・ペイン）を対象に実証運用を行います。

## 2. 看護部への依頼事項

以下の4点について、ご協力をお願いいたします。

1. **ベッドコントローラー担当者の選定**（1-2名）
   - 6F病棟からアプリ操作の主担当者をご指名ください
2. **毎日約20分の運用時間確保**（朝会前後15分＋夕方データ入力5分）
   - 朝会前：ダッシュボード確認（5分）、朝会後：What-if検証（5分）
3. **日次データ入力**（夕方、所要約5分）
   - 病床マップへの在院日数入力
4. **週1回のフィードバック**
   - 使い勝手・改善点についてのご意見（口頭またはメモで可）

## 3. アプリでできること（看護部にとってのメリット）

- **朝会での「今日の病棟状態」を30秒で把握**
  - 意思決定スコアにより、一目で病棟の状態（安全/注意/要対応）がわかります
- **退院調整の判断根拠が数値化される**
  - これまでの感覚的判断が、稼働率・運営貢献額への影響度として「見える化」されます
- **What-if分析で退院・入院の影響を事前検証**
  - 「今日退院させた場合」「追加入院を受けた場合」の稼働率・運営収支変動をシミュレーションできます
- **ベッドコントローラーの役割が明確化**
  - アプリ上で「オーケストレーター」としての判断ポイントが整理されており、役割の標準化に寄与します
- **HOPE電子カルテのToDoへサマリーを直接送信**
  - アプリのボタン操作で、2種類のサマリーをHOPEのToDo欄に送信できます
    - **全体サマリー（朝の申し送り用）:** 病棟稼働状況・本日退院予定・空床数などを一覧化
    - **医師別退院調整依頼:** 担当医ごとの退院候補患者リスト・在院日数アラートを個別送信
  - HOPE ToDo欄の400文字制限に自動対応するため、手動で要約する手間がありません
  - 院内LAN完結で動作し、外部通信は発生しません
