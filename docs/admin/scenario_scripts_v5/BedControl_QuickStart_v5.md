# ベッドコントロール v5 クイックスタート

- **対象:** 試験運用 1 週目の医師・看護師・師長向け
- **読む時間:** 3 分
- **目的:** 画面を開いた時に、今日あと何をすればよいか迷わないようにする

---

## まず 30 秒で見る場所

1. **稼働率**
2. **空床**
3. **在院患者数**
4. **最上段の行動フォーカスカード**

詳細サマリーは、最初から全部読まなくて大丈夫です。必要な時だけ開きます。

---

## 今日の一手の読み方

行動フォーカスカードは、各セクションの一番上にあります。

| 表示 | 意味 |
|---|---|
| title | 今日の判断テーマ |
| action | 今日やること |
| chips | 見る数字を 3 つに絞ったもの |
| note | 安全上の前提、制度上の注意 |

赤いカードは今日対応、黄色は要注意、緑は維持です。

---

## 6F 看護必要度を見るとき

6F は、入院数を減らす提案ではありません。
入院数はこれまでの実績ペースを前提に、**適応のある治療・ケア・記録の取りこぼしを防ぐ** 方針です。

### 医師が見ること

- この患者に医学的に適応のある処置・薬剤・管理があるか
- A6、C 項目、輸血、酸素、シリンジポンプなどが記録と一致しているか
- 開始日、終了予定、継続理由が分かるか

### 看護師が見ること

- 今日の状態と記録が一致しているか
- 酸素、点滴、輸血、創傷処置、救急搬送後などが漏れていないか
- 医師確認が翌日持ち越しになっていないか

---

## 疾患別マニュアルの使い方

1. サイドバーの `📚 看護必要度クイックリファレンス` を開く
2. `🏥 疾患別マニュアルを開く` を押す
3. 疾患名や処置名で検索する
4. まず `最初に見る` タブだけ読む
5. 必要時だけ `評価・記録`、`具体例`、`根拠・安全` を見る

検索例:

- `心不全`
- `酸素`
- `輸血`
- `PTCD`
- `PEG`
- `ペイン`

---

## 絶対にしないこと

- 看護必要度のために不必要な処置を追加する
- 評価表や記録を事実と違う内容で書く
- 病棟都合で患者を選別する
- 件数ペースをノルマとして扱う

迷ったら、患者さんに医学的な適応があるかを先に確認します。

---

## セクション別の使い分け

| セクション | 使う場面 |
|---|---|
| `📊 今日の運営` | 朝いちばん。稼働率、空床、在院患者数、今日の一手 |
| `🔮 What-if・戦略` | 条件を 1 つだけ動かして、何人で変わるか確認 |
| `🛡️ 制度管理` | 救急 15%、LOS、C 群など制度リスク確認 |
| `🏥 退院調整` | 退院日を確定する患者、家族・施設調整 |
| `📈 過去1年分析` | 6F 必要度、月間入院数、過去実績の確認 |
| `⚙️ データ・設定` | 日次入力、医師別分析、HOPE 送信、エクスポート |

## 1 週間の試験運用で見たいこと

1. 今日の一手が分かったか
2. 数字の意味が分かったか
3. 危ない処置を促されたと感じなかったか
4. 疾患別マニュアルを実際に使えたか

フィードバックは [試験運用フィードバックアンケート](../pilot_feedback_survey_2026.md) に記入してください。

---

## 困った時

| 困りごと | 対応 |
|---|---|
| 画面の数字が古そう | `⚙️ データ・設定` で日次入力を確認 |
| どこを見ればよいか分からない | 最上段の行動フォーカスカードに戻る |
| 看護必要度の項目が分からない | 疾患別マニュアルを検索 |
| 適応外に見える表現がある | その場で止め、副院長へ共有し、アンケートにも記載 |
| 操作に迷う | まず `📊 今日の運営` に戻る |
