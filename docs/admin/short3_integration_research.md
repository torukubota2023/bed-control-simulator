# 短期滞在手術等基本料3（短手3）— 制度リサーチとアプリ組み込み案

**作成日:** 2026-04-08
**対象アプリ:** ベッドコントロールシミュレーター
**ステータス:** 設計案（実装未着手）

---

## 1. 基本的な仕組み

**短期滞在手術等基本料3（通称「短手3」）** は、4泊5日以内の短期入院で行う特定の手術・検査を**包括点数**で評価する制度です。

### 制度のポイント

| 項目 | 内容 |
|---|---|
| **対象期間** | 入院日から **5日以内**（4泊5日） |
| **算定方式** | 包括点数（入院料・投薬・注射・検査・画像・術後管理まで全て込み） |
| **対象手術** | 73項目（内視鏡的大腸ポリープ切除術、水晶体再建術、鼠径ヘルニア手術、下肢静脈瘤手術、痔核手術 等） |
| **DPC病院** | 従来は算定不可（**2026年改定で一部変更予定**） |
| **当院への影響** | 当院はDPC病院ではない前提で算定可 |

### 代表的な点数（令和6年度改定）

- **K721 内視鏡的大腸ポリープ・粘膜切除術 長径2cm未満**: **12,580点**（12万5,800円）
- **長径2cm以上**: **16,153点**（16万1,530円）
- 4泊5日までの入院料・検査・薬剤すべて含めた包括点数

---

## 2. 【最重要】平均在院日数計算への影響

**短手3算定患者は、入院5日以内の期間は平均在院日数計算から完全に除外されます。**

具体的には：

- **分子の在院患者延日数から除外** ← その日の「在院患者数」にカウントしない
- **分母の新入院・新退院から除外** ← 入院数・退院数にカウントしない
- **6日目以降に残った場合**: その日から計算対象に復帰（入院日起算でカウント）

同様に **看護必要度・日常生活機能評価** からも5日以内は除外されます。

---

## 3. 地域包括医療病棟での意味合い

### メリット

1. **高単価**: 大腸ポリペクトミー2cm未満で 12,580点/4泊5日 = **約25,000円/日** の包括収入
2. **高回転**: 週単位で入退院、予定通りに動く
3. **病床稼働率には貢献**: ベッドは使っているので稼働率の分母・分子には入る

### ⚠️ 重要な落とし穴（2026年改定）

**短手3 を平均在院日数計算から除外すると、施設基準判定は厳しくなる方向に動く**

- 短手3 は超短期入院（平均約2日）なので、全体平均（約18-21日）より大幅に短い
- これを分母・分子から除外すると、**平均は「増える」方向に変化**する
- 分母（新入院+退院）の減少率 > 分子（在院延日数）の減少率 → 平均値は上昇

実測例（デモデータ5F、短手3=40件を加算した場合）:
- 通常 rolling LOS: 18.7日
- 短手3除外後: **23.0日** (+4.3日)

**この差分は施設基準上限（20〜21日）をまたぐ可能性があり、経営判断上極めて重要**

### 注意点

1. **2026年改定で点数引下げの方向性**: 外来シフトを促すため、短手3の点数が下がる可能性
2. **A群との混同に注意**: 一般の1-5日目入院患者（A群）とは **運営貢献額の計算が別**
3. **運営貢献額の扱い**: 包括点数なので日数按分ではなく「入院1件あたりの固定収入」で扱うべき

### 現状の計算と齟齬がある箇所

## 5. アプリへの組み込み提案（A群内数方式）

### 重要な洞察: 短手3期間 = A群期間（完全一致）

| 項目 | A群 | 短手3 |
|---|---|---|
| 対象期間 | 入院1-5日目 | 入院日から5日以内（4泊5日） |
| **6日目以降** | B群へ自動移行 | 短手3として算定不可になり、通常入院料へ |

**→ 短手3患者は、必ず A群の期間内で退院する**

この一致を利用すると、新カテゴリ（S群）を作らず、**A群の内数として管理**できる。

### 設計原則

- **入力者の負担を最小化** — 既存入力フローに1項目追加のみ
- **既存の入力方針踏襲** — プルダウン・スライダー・数値入力のみ
- **自動計算を最大化** — ユーザーは「数」だけ入力、分類は自動
- **退院時の区別は不要** — 入院時にフラグ付けするだけ、退院は5日以内で自動

### 日次入力フロー（最終形）

| # | 項目 | 操作 | 例 |
|---|---|---|---|
| ① | 病棟 | プルダウン | 5F |
| ② | 日付 | カレンダー | 今日 |
| ③ | 在院患者総数 | 数値 | 42 |
| ④ | 新規入院数 | 数値 | 5 |
| ⑤ | 退院人数 | 数値 | 3 |
| ⑥ | 各退院者の在院日数 | スライダー | 5/10/18日 |
| ⑦ | 備考 | テキスト | — |
| **⑧** | **うち短手3（内数）** | **数値** | **2** |

**所要時間: 従来5分 → 5分5秒**（+5秒のみ）

### データ構造と計算ロジック

#### 1. 新規フィールド（CSVに1列追加のみ）

```csv
date, ward, total_patients, new_admissions, discharges, ...
  new_admissions_short3,   ← 新規追加（1列だけ）
...
```

#### 2. 短手3スライドバッファ（内部状態）

```python
short3_buffer = {
    "day_1": 2,  # 今日入院した短手3
    "day_2": 1,  # 昨日入院した短手3
    "day_3": 2,
    "day_4": 0,
    "day_5": 1,  # 今日退院予定
}

# 現在在院中の短手3患者数 = 合計 = 6名
short3_in_hospital = sum(short3_buffer.values())

# 今日の短手3退院数 = day_5 の値 = 1名
short3_discharges_today = short3_buffer["day_5"]
```

#### 3. 毎日のシフト処理

```python
def advance_short3_buffer(prev_buffer, new_s3_admissions):
    return {
        "day_1": new_s3_admissions,
        "day_2": prev_buffer.get("day_1", 0),
        "day_3": prev_buffer.get("day_2", 0),
        "day_4": prev_buffer.get("day_3", 0),
        "day_5": prev_buffer.get("day_4", 0),
        # prev_buffer["day_5"] は今日退院したので消滅
    }
```

#### 4. 平均在院日数計算（自動で短手3除外）

```python
def calculate_avg_los_excluding_short3(df):
    # 在院患者延日数から短手3在院日数を差し引く
    total_pd = df["total_patients"].sum() - df["short3_in_hospital"].sum()
    # 新入院・退院からも短手3分を差し引く
    total_adm = df["new_admissions"].sum() - df["new_admissions_short3"].sum()
    total_dis = df["discharges"].sum() - df["discharges_short3_auto"].sum()
    ht = (total_adm + total_dis) / 2
    return total_pd / ht if ht > 0 else 0
```

`discharges_short3_auto` は**スライドバッファから自動計算**するので、ユーザー入力は不要。

#### 5. A群人数の内訳管理

```python
# A群全体は既存計算のまま
phase_a_total = daily_adm × 5  # or 既存ロジック

# その内訳として短手3分を表示
phase_a_short3 = sum(short3_buffer.values())
phase_a_regular = phase_a_total - phase_a_short3
```

## 6. 段階的な導入プラン（推奨）

混乱しないよう **3段階** で進める：

### Phase 1: 最小機能（入力のみ）

- CSVに1カラム追加（`new_admissions_short3`）
- 日次データ入力タブに⑧フィールド追加
- 内部的に記録するだけ、計算には反映しない
- **既存の動作は何も変わらない**（安全）

### Phase 2: 計算分岐

- スライドバッファを実装
- 平均在院日数（集計・rolling）を短手3除外版で計算
- 本日の病床状況・意思決定ダッシュボードに「(短手3除外後)」表記追加
- A群の内訳として「うち短手3」を表示

### Phase 3: 収益計算の精緻化

- 短手3の包括点数を別計算
- 運営貢献額タブで通常分と短手3分を分離表示
- What-if シミュレーションで「月の短手3件数を変えたら」を試算可能に

## 7. 旧案 vs 新案 比較

| 項目 | 旧案（S群新設） | **新案（A群内数）** |
|---|---|---|
| データ入力 | +2項目（入院・退院両方） | **+1項目（入院のみ）** |
| UI変更 | 新カテゴリ追加で複雑化 | **既存A/B/C構造を維持** |
| 退院時の分類 | 必要 | **不要（自動処理）** |
| データ構造 | CSVに2カラム追加 | **CSVに1カラム追加** |
| 既存ロジックへの影響 | フェーズ計算を拡張 | **内数として差し引きのみ** |
| 視覚的理解のしやすさ | S群の説明が必要 | **「A群のうち短手3」で直感的** |

## 8. 当院の実態（2026-04 ヒアリング結果）

### 確定情報

| 項目 | 当院の数値 |
|---|---|
| **月間件数** | **20〜25件/月** |
| **主な術式** | 大腸ポリペクトミー **80%**（16〜20件）、鼠径ヘルニア手術、ポリソムノグラフィー検査 |
| **運用病棟** | **5F・6F 両方**で実施 |

### 数値の含意（Little 法則）

- 1日あたり入院 ≈ 22 ÷ 30 = **約 0.73 件/日**
- 平均在院日数 ≈ 4 日（5日以内）
- 病院全体で常時 **約 2.9 人**が短手3患者として在院中
- 各病棟で常時 **約 1.5 人**ずつ（半々と仮定）

### Phase 2以降で必要になる可能性

4. **現状の記録方法**: 短手3患者は現在どのように病棟で管理されていますか？（Phase 1 では不要）
