# 🏥 退院集中対策 一枚要約（病棟詰所掲示用）

**おもろまちメディカルセンター / 2026-04-24 / 副院長 久保田 徹**

---

## なぜ「退院の日」を揃えないほうがいいのか — 3 行で

> **過去 1 年で退院集中 49 回（2 週に 1 回）** → **稼働率 0.5〜0.9 pt 低下** → **年間 812 万円の機会損失**

---

## 📊 データが示す事実（過去 1 年 1,793 件）

| 病棟 | 枠超過日数 | 集中する曜日 | 最大退院数/日 |
|------|----------|-------------|---------------|
| **5F** | 22 日 / 382 日 (5.8%) | **土曜日 8 日**（36%） | 8 名 (5/29, 7/19, 3/3) |
| **6F** | 27 日 / 387 日 (7.0%) | **火曜 8 日・金曜 8 日**（59%） | 9 名 (12/30, 1/27) |

**「今日は 8 名退院」は最大級ではなく「よくある重なり」。**

---

## 💰 病院経営への影響

- **稼働率**: 枠制限なしで +0.51 pt（5F）、+0.90 pt（6F）の改善余地
- **診療報酬**: 地域包括医療病棟 3,367 点/日（33,670 円/床日）
- **年間効果**: 5F 約 293 万円 + 6F 約 519 万円 = **合計 812 万円/年**
- 実運用で 50〜70% 実現できれば **400〜570 万円/年**

## 🎯 現仕様で実装済みの対策

| 機能 | 効果 |
|------|------|
| 月〜土 5 枠 / 日祝 2 枠の基本枠ルール | 1 日の退院上限を可視化 |
| 超過時は翌営業日を -N 自動縮小 | 累積しない（翌々日は 5 枠復帰） |
| 日曜退院を推奨（⭐ マーカー） | 空いている枠の活用 |
| 動的枠調整（稼働率 95% 以上で +1） | 繁忙期の柔軟性 |
| 主治医別突発退院頻度の可視化 | 運用ルール周知の対象を絞る |

**シミュレーション: 枠制限 UI で年間 49 回の超過日を「全てゼロ」に抑制**

## 🏥 現場の 5 つのメリット

1. 🩺 **病棟業務の平準化** — 退院処理 8 名 → 5 名で手続き・送迎が分散
2. 👩‍⚕️ **師長の裁量確保** — スタッフ配置の急遽変更が不要
3. 📅 **入退院調整 NS の業務リズム** — 週後半に負荷集中しない
4. 💼 **経営部の予見性** — 稼働率急落を事前予測
5. 🛡️ **コントローラー不在でも事故なく運営** — アプリが判断材料を提示

## 📅 運用フロー（1 週間サイクル）

```
水曜夜    → カンファ前「混雑警告」自動サマリー（リスク日 TOP 3）
木曜カンファ → 退院予定を確定、枠超過時は別日候補を検討
木金     → カレンダーで確認・微調整、日曜推奨枠の活用
退院当日   → 突発退院を ⚡ マーカーで監視、主治医別集計
月初     → 月次レポートを経営会議に自動提出
```
