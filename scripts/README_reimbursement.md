# 地域包括医療病棟入院料 診療報酬シミュレーター

## 1. 概要

本アプリは、**令和8年度（2026年度）診療報酬改定**に基づく地域包括医療病棟入院料の収益シミュレーターである。

病棟の入院患者構成（ケースミックス）を入力すると、6区分の点数テーブルに基づいて月間・年間の収益を自動計算し、施設基準の充足状況をリアルタイムでチェックする。感度分析やWhat-ifシミュレーションにより、救急割合・在院日数の変動が収益に与える影響を可視化できる。

### 対象ユーザー

- 病院経営者（理事長・院長・副院長）
- 事務長・医事課スタッフ
- 病棟管理者（看護師長・看護部長）

### 既存ベッドコントロールシミュレーターとの違い

| 項目 | ベッドコントロールシミュレーター | 診療報酬シミュレーター（本アプリ） |
|------|-------------------------------|----------------------------------|
| 目的 | 稼働率90-95%維持と粗利最大化 | 入院料の点数区分別収益シミュレーション |
| 視点 | 患者フェーズ（A/B/C群）のベッド占有 | ケースミックス（診療科×入院形態×手術有無） |
| 主な出力 | 日次ベッド占有状況、フェーズ構成比 | 6区分点数計算、施設基準チェック、感度分析 |
| 入力 | 入退院パターン、稼働率目標 | ケースミックス、加算設定、施設条件パラメータ |
| 活用場面 | 日常のベッド運用最適化 | 診療報酬改定対応、経営戦略策定、施設基準モニタリング |

両者は補完関係にあり、ベッドコントロールシミュレーターの `PHASE_COSTS` 定数は本アプリの `reimbursement_config.py` から連携できる設計になっている。

## 2. 前提条件

- **令和8年度（2026年度）診療報酬改定**の告示内容に基づく
- 地域包括医療病棟入院料1/2の**6区分**点数テーブルを実装（3,066〜3,367点）
- **おもろまちメディカルセンター**（総病床数94床、2病棟各47床、月間入院数約150件）をデフォルト設定として搭載
  - 5F: 外科・整形外科（手術・リハ中心）
  - 6F: 内科・ペイン科（内科系・疼痛管理中心）
  - A100一般病棟入院基本料の算定病棟なし → 入院料1がデフォルト
- 1点 = 10円で換算

## 3. 入力項目

### A100一般病棟の有無（必須）

- **なし（入院料1）**: A100急性期一般入院基本料を算定する病棟がない病院 → 高点数（3,117〜3,367点）
- **あり（入院料2）**: A100を算定する病棟がある病院 → やや低点数（3,066〜3,316点）

### ケースミックス（病棟×科×入院形態×手術有無）

各行は以下の属性を持つ:

| 項目 | 選択肢 | 説明 |
|------|--------|------|
| 病棟 | 5F / 6F | 入院先の病棟 |
| 診療科 | 内科 / ペイン科 / 整形外科 / 外科 | 主科 |
| 入院形態 | 救急 / 予定 | 救急搬送後入院か予定入院か |
| 手術有無 | あり / なし | 主傷病に対する手術の有無 |
| 月間件数 | 0〜200 | 当該組み合わせの月間入院件数 |
| 平均在院日数 | 1.0〜90.0 | 当該組み合わせの平均在院日数 |

入院料区分と基本点数は入力値から自動計算される。行の追加・削除も可能。

### 加算設定（トグル可能、相互排他あり）

| カテゴリ | 加算名 | 点数/日 | 適用期間 | デフォルト |
|----------|--------|--------:|----------|-----------|
| 基本加算 | 初期加算 | 150 | 1〜14日 | ON |
| リハ・栄養・口腔連携 | 連携加算1 | 110 | 1〜14日 | ON |
| リハ・栄養・口腔連携 | 連携加算2 | 50 | 1〜14日 | OFF |
| 看護補助体制 | 25対1 | 240 | 1〜14日 | OFF |
| 看護補助体制 | 50対1 | 160 | 1〜14日 | OFF |
| 夜間看護 | 50対1 | 125 | 全期間 | OFF |
| 夜間看護 | 100対1 | 105 | 全期間 | OFF |
| 看護職員夜間配置 | 12対1 | 110 | 1〜14日 | OFF |
| 看護職員夜間配置 | 16対1 | 45 | 1〜14日 | OFF |
| 物価対応 | 令和8年度 | 49 | 全期間 | ON（年度連動） |
| 物価対応 | 令和9年度 | 98 | 全期間 | OFF（年度連動） |

同一カテゴリ内の加算は**相互排他**（1つのみ選択可）。物価対応料は年度設定で自動切替。

### 施設基準パラメータ

サイドバーで以下を設定:

- 85歳以上割合（%）: スライダー 0〜50%、デフォルト25%
- ADL低下割合（%）: 数値入力、デフォルト3.0%
- 在宅復帰率（%）: 数値入力、デフォルト80.0%
- 看護必要度割合（%）: 数値入力、デフォルト20.0%
- データ提出加算: チェックボックス、デフォルトON
- リハ専門職配置（名）: 数値入力、デフォルト2名

## 4. 点数ロジック

### 6区分の点数テーブル（1日につき）

| 病棟入院料 | 入院料区分 | 入院形態 | 手術 | 点数 |
|-----------|-----------|----------|------|-----:|
| 入院料1（A100なし） | 入院料1 | 救急 | なし | 3,367 |
| 入院料1（A100なし） | 入院料2 | 救急/予定 | あり/なし | 3,267 |
| 入院料1（A100なし） | 入院料3 | 予定 | あり | 3,117 |
| 入院料2（A100あり） | 入院料1 | 救急 | なし | 3,316 |
| 入院料2（A100あり） | 入院料2 | 救急/予定 | あり/なし | 3,216 |
| 入院料2（A100あり） | 入院料3 | 予定 | あり | 3,066 |

### 入院料1/2/3の判定ロジック

入院形態と手術有無の組み合わせで自動判定される:

| 入院形態 | 手術有無 | 入院料区分 | 理由 |
|----------|---------|-----------|------|
| 救急搬送 | なし | **入院料1**（最高点） | 救急対応は予測不能で負担大、手術収入なし |
| 救急搬送 | あり | **入院料2** | 救急だが手術料は出来高で別途算定可 |
| 予定入院 | なし | **入院料2** | 計画的だが手術収入なし |
| 予定入院 | あり | **入院料3**（最低点） | 計画的かつ手術料は出来高で算定可 |

### 加算の適用期間

- **1〜14日限定**: 初期加算、リハ・栄養・口腔連携加算、看護補助体制加算、看護職員夜間配置加算
- **全期間（1日目〜退院まで）**: 夜間看護補助体制加算、物価対応料

15日目以降は14日限定の加算が消失し、1日あたりの収入が大幅に減少する。

### 日次点数の計算式

```
日次点数(day) = 基本点数(ward_type, tier) + Σ 適用加算の点数(day)
```

ただし day > 90 の場合:
```
日次点数(day) = 988 + Σ 適用加算の点数(day)
```

### 90日超の逓減

入院が90日を超えると基本点数が**988点/日**（地域一般入院料3相当）に激減する。約3,000点台から988点への低下であり、収入が約1/3.5になる。

### 収益計算

```
1件あたり総点数 = Σ(day=1..LOS) 日次点数(day)
1件あたり収益（円） = 総点数 × 10（円/点）
月間収益 = 1件あたり収益 × 月間件数
年間収益 = 月間収益 × 12
```

## 5. 施設基準（制約条件）

### MUST（必須 - 満たさないと算定不可）

| 基準 | 閾値 | 演算子 | 備考 |
|------|------|--------|------|
| 平均在院日数 | 20日 | <= | 85歳以上割合20%ごとに+1日緩和、最大24日 |
| 救急搬送後患者割合 | 15% | >= | 入院患者に占める救急搬送後患者の割合 |
| ADL低下割合 | 5% | < | 退院時ADLが入院時より低下した患者の割合 |
| 在宅復帰率 | 72.5% | >= | 自宅等に退院した患者の割合 |
| 重症度・医療看護必要度 | 16% | >= | 必要度I or IIの該当患者割合 |
| データ提出加算 | 届出済 | >= | DPCデータの提出体制 |

### SHOULD（推奨 - 満たさないと警告）

| 基準 | 閾値 | 備考 |
|------|------|------|
| リハ専門職配置 | 1名以上 | PT/OT/STの病棟配置 |

### 85歳以上割合による緩和措置

85歳以上の患者割合が高い病院では、一部の基準が自動緩和される:

- **平均在院日数**: 基本20日 + floor(85歳以上割合 / 20%) 日（最大24日）
  - 例: 85歳以上割合25% → 20 + 1 = 21日まで許容
  - 例: 85歳以上割合45% → 20 + 2 = 22日まで許容

│   - Tab 1: ケースミックス入力（data_editor）

## 7. 起動方法

### 前提パッケージ

```bash
pip install streamlit pandas plotly
```

### 起動

```bash
streamlit run scripts/reimbursement_simulator_app.py
```

ブラウザが自動で開き、ダッシュボードが表示される。サイドバーで届出区分・加算・施設条件を設定し、各タブで結果を確認する。

## 9. 検証方法

### ユニットテストで検証する項目

- **点数区分テスト**: 6区分の全組み合わせ（WardType x AdmissionTier）で正しい点数が返ること
- **日次点数テスト**: 14日目と15日目の境界で加算の適用/非適用が正しく切り替わること
- **90日超テスト**: 91日目以降の基本点数が988点に逓減すること
- **施設基準判定テスト**: 各制約条件の閾値境界で passed/failed が正しく判定されること
- **85歳以上緩和テスト**: 高齢者割合に応じた平均在院日数の閾値調整が正しいこと

### 手計算との照合例

**条件**: 入院料1（A100なし）、救急搬送・手術なし、在院日数17日、初期加算+リハ連携加算1+物価対応料

```
1〜14日目: 3,367 + 150 + 110 + 49 = 3,676点/日 × 14日 = 51,464点
15〜17日目: 3,367 + 49 = 3,416点/日 × 3日 = 10,248点
合計: 61,712点 = 617,120円
```

この値がシミュレーターの出力と一致することを確認する。
